## Домен и канонические URL

> **ЕДИНЫЙ продакшн-домен: `odaeda.ru`** — РЕАЛЬНЫЙ прод-домен NiloV Catering (ИП Нилов), подтверждён заказчиком (2026-07-20). Зафиксирован как канон для всего сайта (P0-4, устранение рассинхрона canonical).

- **Продакшн:** `odaeda.ru` — единственный публичный домен. На него указывают ВСЕ
  `canonical` / `metadataBase` / `sitemap.xml` / `robots.txt`.
- **Допустимый alias:** `nilov-catering.ru` — может использоваться как редирект (301) на `odaeda.ru`
  при необходимости, но canonical ВСЕГДА на `odaeda.ru`.
- **Staging / preview:** `*.vercel.app` — ТОЛЬКО для предпросмотра деплоев (Vercel preview).
  Не индексируется: `<head>` несёт `noindex`, canonical ведёт на прод
  (`https://odaeda.ru/...`); в `robots.txt` на `*.vercel.app` — `Disallow: /`.
- **Локальная разработка:** `localhost` — НЕ должен фигурировать в прод-конфигах.

| Назначение | Значение |
|---|---|
| `metadataBase` | `https://odaeda.ru` |
| canonical (каждая страница) | `https://odaeda.ru<route>` |
| `sitemap.xml` | `https://odaeda.ru/sitemap.xml` |
| `robots.txt` | `https://odaeda.ru/robots.txt` |
| `llms.txt` | `https://odaeda.ru/llms.txt` |

> **Правило кода (`app/*`):** canonical и `metadataBase` ДОЛЖНЫ ставиться на `odaeda.ru`,
> а НЕ на `*.vercel.app` / `localhost`. Staging-превью (`*.vercel.app`) получает
> `noindex` + canonical на прод, чтобы не дублировать контент в выдаче.

## Каноничная иерархия меню (против SEO-каннибализации)

4 пути просмотра меню разведены по ИНТЕНТУ — дубли не пересекаются, каннибализации нет:

| Роут | Назначение | Интент |
|---|---|---|
| `/menu` | Каталог меню (ленд + входы во все форматы) | «посмотреть всё меню» |
| `/menu/{format}` | `{format}` ∈ `furshet`·`banquet`·`coffee-break` — пакеты (PackageGrid) | «меню под формат» |
| `/menu/catalog` | Блюда поштучно (фильтр по категориям/аллергенам) | «найти конкретное блюдо» |
| `/menu/show-cooking` | Интерактивные станции (ShowCookingGrid) | «живые станции» |
| `/plan` | Хаб инструментов (калькулятор + конструктор) | «рассчитать / собрать» |
| `/plan/calculator` | Калькулятор стоимости | «узнать цену» |
| `/plan/constructor` | Конструктор меню (мастер 6 шагов) | «собрать своё» |
| `/seasonal` | Сезонный модуль ОТДЕЛЬНО (ББQ / НГ / Масленица) | «сезонное предложение» |

Правила:
- CTA `MenuPreview` ведёт в `/menu/{format}` (НЕ в `/menu/{collection}` — такого роута нет).
- `/seasonal` — самостоятельный роут, НЕ дублирует новогодние/летние офферы из `/events`/`/menu`;
  он агрегирует их по сезону, но живёт отдельно, чтобы не каннибализировать посадочные `/events/*` и `/menu/*`.

## Правило для меню (из blueprint, но упрощаем язык)
Клиент думает «мне нужна свадьба на 80 человек», а не «фуршет Эконом/Премиум/Люкс».
Поэтому на `/events/*` сначала контекст события → потом рекомендованные форматы из `/menu`.
Конструктор меню = мастер, а не каталог с галочками (см. `03_JOURNEYS.md`).
На `/plan/*` и `/contact` показываем `AvailabilityCalendar` (urgency «даты заполняются») — доверие к исполнению.
