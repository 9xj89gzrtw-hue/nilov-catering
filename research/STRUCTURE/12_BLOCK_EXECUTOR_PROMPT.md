# ПРОМПТ-ШАБЛОН: Исполнитель блока (subagent)

**Назначение:** копируется в `delegate_task` (или отдельный чат сборки) для сборки ОДНОГО блока.
Подставь `{{BLOCK_NAME}}`, приложи нужные файлы из `research/STRUCTURE/` — и субагент строит блок
изолированно, строго по дизайн-системе. НЕ придумывает дизайн сам.

---

## Шаблон (копировать ниже `---`)

```
Роль: Senior Frontend Engineer (Next.js 16 + React 19 + Tailwind v4 + TypeScript).
Задача: реализовать ОДИН переиспользуемый блок "{{BLOCK_NAME}}" для сайта NiloV Catering.

НЕ выходи за рамки блока. НЕ собирай страницу целиком. НЕ придумывай визуальный стиль —
стиль жёстко задан Design Tokens. Любое решение по цвету/типографике/анимации бери ТОЛЬКО из приложенных документов.

## Входные документы (источник истины)
- 06_TOKENS.md        — цвета, типографика, радиусы, тени, motion-tokens (ОБЯЗАТЕЛЬНО)
- 01_VISUAL_DNA.md    — визуальное направление (light/airy), motion-language
- 04_BLOCKS.md         — описание блока {{BLOCK_NAME}} и примитивов
- 09_HERO_STORYBOARD.md (если блок Hero) / 10_FORMAT_SHOWCASE_STORYBOARD.md /
  11_GALLERY_TEASER_STORYBOARD.md (если соответствующий блок) — раскадровка
- 03_JOURNEYS.md       — контекст CTA и конверсии (для ссылок/действий)

## Требования к реализации
1. Технологии: Next.js 16 App Router, RSC по умолчанию; 'use client' ТОЛЬКО если нужен интерактив
   (Framer Motion/Lenis/hover-state). Импорт токенов через Tailwind utilities (bg-background, text-foreground,
   bg-primary, text-gold, font-heading…) или var(--color-*). ХАРДКОД HEX в компоненте = нарушение.
2. Анимации: использовать ТОЛЬКО motion-токены из 06_TOKENS.md. Reveal через Framer Motion
   `whileInView` (initial y:24/opacity:0 → whileInView y:0/opacity:1, stagger 80ms, ease-out-expo, viewport once).
   Hover — Ken Burns / magnetic / tilt строго как в раскадровке блока. Уважать useReducedMotion() (off).
3. A11y (WCAG 2.2 AA): семантика (section/article/nav), focus-visible 2px gold, label на input,
   aria на интерактив, alt на все img, touch-targets ≥44px, skip-link глобально.
4. Mobile-first: брейкпоинты sm640/md768/lg1024/xl1280. На mobile — упрощённая анимация (без parallax/tilt).
5. Производительность: lazy-load изображений (next/image), blur-up, skeleton; не блокировать LCP.
6. Контент: placeholder-тексты/цены БРАТЬ из приложенных документов (не выдумывать). Ссылки CTA —
   из 02_IA.md (например, /menu/furshet). Никаких мёртвых #ссылок.
7. Структура файлов: компонент в `components/sections/{{BLOCK_NAME}}.tsx`
   (папка `sections/` УЖЕ существует и содержит готовые секции — НЕ создавать `blocks/`!
   см. `05_BUILD` Фаза 0 п.3 и `41_BUILD_CHECKLIST` Шаг 1). Примитивы — из `components/ui/*`
   (не дублировать Button/Card/Container — использовать готовые из токенов).
   Перед сборкой сверить имя блока `04_BLOCKS.md` с реальным файлом через `39_CANON_INDEX.md`
   (многие блоки `04` уже реализованы под другими именами: TrustBar/AwardsStrip→`SocialProofBar`,
   EventTypeSelector→`ServicesSection`, WhyUs→**дописать из `PhilosophySection` (УЖЕ ЕСТЬ orphan, не создавать новый `WhyUs.tsx` — см. `39:64`/`41:65`/`05:23,53`)**.

## Что НЕ делать
- Не менять палитру/типографику под своё усмотрение.
- Не добавлять функции сверх описания блока (без подключённой AI-модели «Консьерж», без AR и т.п. — это резерв).
- Не писать «красивый заглушечный» код с серыми плитками — фото/видео через next/image со
  placeholder-URL (например, /placeholder/{{BLOCK_NAME}}-1.webp), чтобы при сборке подставить реал.

## Формат вывода
- Путь и полный код файла(ов) блока.
- 5-строчный отчёт: что сделано, какие токены/примитивы использованы, какие edge-кейсы учтены,
  что оставлено как placeholder.
- Самооценка по чек-листу из раскадровки блока (что выполнено / что нет).

## Критерий успеха
Блок визуально и по поведению совпадает с раскадровкой, собирается без ошибок типов,
проходит чек-лист «дорого» из раскадровки. Критик (отдельный субагент) проверит результат,
а не промежуточный код.
```

## Как использовать (поток сборки)
1. Прочитать `39_CANON_INDEX.md` (мост спека↔код) + `41_BUILD_CHECKLIST.md` (статус блока + гейты).
2. Выбрать блок из `04_BLOCKS.md` по фазе (`05_BUILD.md`).
3. Создать задачу субагенту: вставить шаблон выше, подставить `{{BLOCK_NAME}}`,
   приложить `06_TOKENS.md` + `01_VISUAL_DNA.md` + `04_BLOCKS.md` + нужную раскадровку + `02_IA.md`/`03_JOURNEYS.md`.
4. Получить код блока → собрать в страницу.
5. Отдать результат **независимому критику** (другой субагент, роль «art-director luxury catering»)
   с раскадровкой и чек-листом — он НЕ видел промежуточный код.
6. Правки по его отчёту → следующий блок.

## Роли (резюме, на наши реальные инструменты)
- **Архитектор:** я (Hermes) + папка STRUCTURE как истина.
- **Исполнитель:** субагент (leaf) по этому шаблону, 1 блок = 1 контекст.
- **Критик:** независимый субагент, только результат + раскадровка + чек-лист.
- **Финальный аудит:** 3 параллельных (creative/UX/frontend) → сводка.
