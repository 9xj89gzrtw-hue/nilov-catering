## 1. Почему фото/видео — это не «картинки», а продажа

foodshot.ai (2026): кейтеринг ≠ ресторан. Ресторан продаёт **одно блюдо** на красивой
тарелке. Кейтеринг продаёт **опыт события целиком** — клиент должен увидеть СВОЙ банкет:
линию буфета с паром, коктейль-час с пассерами, сервировку с цветами. Плохие телефонные
снимки под люминесцентным светом банкетного зала = теряем брони. Проф. фото в портфолио —
ключевой фактор (один шаблон закрыл $16M сделок в 2025).

→ **Для нашего сайта:** каждое фото/видео должно продавать НЕ блюдо, а СОБЫТИЕ.

## 3. ВИДЕО — формат «как в Drinkit» (живые фото по умолчанию)

**Drinkit-паттерн (из `01_VISUAL_DNA`):** все фото «живые и что-то делают» — это не
статика, а **loop-видео/Ken Burns по умолчанию**. На сайте это означает:

- **Hero:** [Волна 13, C-R1] poster (blur-up) = LCP-элемент, показывается мгновенно; поверх — subtle loop-видео/Ken Burns, которое НЕ является LCP-кандидатом (вариант В из `28_HEROCRITIC`, единый контракт с `09`). НЕ full-video hero (вариант Б отвергнут: +3.2s LCP). На slow-connection/`prefers-reduced-motion` → только poster.
- **Карточки меню/форматов:** 5с loop-видео приготовления (повар нарезает, пар с блюда)
  вместо статичного фото — «живое фото», как в приложении заказчика. [Волна 13, C-R3] hover-loop дублируется клавиатурной альтернативой (focus запускает тот же loop; tap-to-play на touch) — WCAG 2.1.1, не только hover.
- **ShowCooking станции:** короткое loop-видео действия (телятина на камне шипит,
  раклет тянется) — вместо фото.
- **VK-лента (ПЕРВИЧНЫЙ эмбед в РФ):** real-time Reels/видео из сообщества VK NiloV — всегда доступна в РФ (Instagram заблокирован). **Instagram (ВТОРИЧНАЯ ссылка):** `@nilov_catering` — fail-soft бейдж «Мы в Instagram → @nilov_catering» (аудитория через VPN). См. канон `04_BLOCKS:275-277,1396-1408`, `29_POSITIONING:54-61`.

**Технические требования к loop-видео (прототип):**
- Длительность 3–6с, **seamless loop** (первый кадр = последний).
- Без звука (muted autoplay). [Волна 13, C-R2] **codec-tiers: AV1 → HEVC (Safari) → H.264/MP4**, `-movflags +faststart` (moov-атом вперёд, −TTFB первого кадра), < 2MB на ролик. `preload="none"` + IntersectionObserver; crossfade poster→video по `canplaythrough` (Mintec 2026). Требование действует для ВСЕХ роликов (Hero И карточки), не только Hero.
- Вертикальный вариант (9:16) для Reels/мобайл-галереи + горизонтальный (16:9) для Hero.
- Светлое, воздушное освещение — НЕ тёмное «кино» (наш DNA, `01`).

**Behind-scenes формула (cynthiaconcierge 2026):** самое вирусное = видео команды,
которая СИНХРОННО сервирует 200 блюд / шеф нарезает / сборка площадки. «Когда планировщик
видит 60с видео вашей команды, что-де-то идеально, — он не просто лайкает, он бронирует».
→ Для нас: 2–3 behind-scenes ролика (сборка, сервировка, шеф) = соц-доказательство + доверие.

## 5. Единый визуальный стиль (чтобы не было «винегрета»)

All Aspects Catering (2026): фото И видео должны быть в **одном стиле** (цветокор,
свет, ракурс) — иначе сайт выглядит собранным из пазлов. Для нас (`01` DNA, светлая):

- **Цветокор:** светлый, тёплый, воздушный. Беж/айвори/золото. НЕ тёмный «кинушный».
- **Свет:** естественный/мягкий студийный. НЕ люминесцентный банкетный (убивает еду).
- **Ракурс:** 45° (еда «смотрит» на зрителя) + top-down (стол/буфет целиком) + macro (деталь).
- **Композиция:** негативное пространство (воздух вокруг), negative space = «дорого».
- **Люди:** живые эмоции гостей + руки команды = доверие (TeamGrid, Testimonials).
