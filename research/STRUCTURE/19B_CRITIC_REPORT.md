## БЛОК 11. LiveInstagramFeed (`04`#11, `16`#8)

**Оси:** Design 5 · UX 5 · Content 5 · Perf 3 · Motion 4 · Responsive 5 · a11y 3 · SEO 3 · Benchmark 5
**Взвешенный итог: 4.30 / 10**
**Вердикт: FAIL** (<6.0)

**Bug-list (спецификация не готова к сборке):**
- **Противоречие реальности.** `04`#11: «real-time лента … (embed Instagram/@nilov_catering)». Но `16`#8 требует «НЕ просто „лента фото“, а **real-time (auto-update)**». Instagram Basic Display API не даёт auto-update embeds — нужен поллинг + токен-инфра + refresh, в spec НЕ описано. Как собирать «real-time» — не指定ано.
- **Perf ≤5 (правило 19#6).** Треть-party Instagram-embed (iframe/script) — известный LCP-киллер. Нет lazy-load, placeholder, стратегии LCP. Нет упоминания LCP/reduced-motion.
- **a11y ≤5.** Встроенный iframe Instagram имеет известные a11y-проблемы (фокус, alt). План a11y отсутствует.
- **SEO 3.** Контент embed НЕ индексируется поисковиком → мёртвый вес для SEO.
- **Нет фолбэка.** Что при недоступности API / приватном аккаунте / rate-limit? Блок может остаться пустым.
- **«Капшены с типом события»** — источник метаданных типа события не определён (где берём «Свадьба»/`«Корпоратив»?).
- **Дизайн.** DNA `01` не цитируется для этой секции; layout/spacing/tokens не заданы.

- Причина массового FAIL: блоки 11–19 описаны в `04` в 1 строку + бенчмарк-абзац в `16`, но **не содержат детального ТЗ** (нет копирайта, нет дата-моделей, нет a11y/perf/reduced-motion, есть противоречия). «Бенчмарк есть» ≠ «готово к сборке».

- **Рекомендация Author:** перед сборкой дописать per-block ТЗ: (1) реальный копирайт/данные, (2) дата-модели (меню, станции, пакеты), (3) a11y + reduced-motion + LCP-стратегии, (4) примирить противоречия (Instagram real-time, EventHero per-type, DishCard zoom-vs-video, Calculator API).
