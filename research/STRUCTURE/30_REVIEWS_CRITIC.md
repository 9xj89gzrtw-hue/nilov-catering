## ПОЧЕМУ (1) НИЗКИЙ — 5 конкретных проблем доверия

1. **Объём = 0 по факту.** «3 500+ событий» в `TrustProof`, но на сайте 3 примера отзывов. По web-data (Sparkmoor/Forbes 2024): товары с >20 отзывами воспринимаются кредибельнее; единичные цитаты = подозрение. 3 шт. — в зоне «не верю».
2. **Примеры выглядят сфабрикованными.** Блок 9 пишет «2–3 РЕАЛЬНЫХ КЕЙСА» с именами/датами/площадками, но это **заготовки `pending-verification`**, а не верифицированные. Имя + дата + «будто мы ничего не делали сами» = классический паттерн фейкового отзыва. Риск бренда.
3. **Пустой `RatingBadge`.** Жёсткое правило G-3: без `url` плитка не рендерится. В спеке `url` НЕ заданы → видимый бейдж внешнего рейтинга отсутствует. Нет соц-доказательства на Google/Яндекс/2ГИС = нет независимой проверки.
4. **Нет микса +/-.** Все примеры — 5/5. По Forbes/TrustYou/LinkedIn: «слишком хорошо, чтобы быть правдой → не верю»; сбалансированный рейтинг (включая пару неровных) вызывает БОЛЬШЕ доверия. Спек не предусматривает показ реальных «неидеальных» отзывов.
5. **Нет ответов компании.** Отзывы без ответа владельца выглядят «купленными/заброшенными». В спеке механизм ответа отсутствует.

## СТРАТЕГИЯ: КАК СДЕЛАТЬ БОЛЬШЕ ДОВЕРИЯ (по web-практике 2026)

**A. Системный сбор (самое важное — его нет).** По Chowly/CloudKitchens: просить в момент пикового удовлетворения + сделать лёгким. Для кейтеринга это **сразу после события** (шаг 5 `ProcessSteps` «Обратная связь»):
   - Авто-триггер (через CRM/CMS `29` B-CMS-5): через 1–2 дня после события → WhatsApp/email/SMS-рассылка с 1 кликом на оставление отзыва.
   - Ссылка ведёт сразу на **внешнюю площадку** (Яндекс/2ГИС/Google) И на внутреннюю форму `/reviews` (фотоотчёт + цитата + разрешение на публикацию видео).
   - Геймификация: «Оставь отзыв → скидка 5% на следующее событие» (B2B-клиенты → бонус менеджеру/следующий кофе-брейк бесплатно).

**B. Видео-отзывы (сильнейший рычаг).** Web-data: 79% потребителей смотрят видео-отзывы; сайты с ними ~+80% конверсии; 76% считают UGC кредибельнее вендорского контента. В спеке УЖЕ есть `TestimonialsCarousel` video-wall + `EventsRecap` — задействовать: на рекап-съёмке просить гостей/организаторов сказать 10–20с в кадр. Это дешевле, чем ставить отдельную съёмку, и выглядит живо.

**C. Интеграция реальных платформ (РФ-специфика).** Для Петербурга: **Яндекс.Карты/Бизнес + 2ГИС — первичны, Google — вторичен** (частично недоступен в РФ). `RatingBadge` (блок 24) это уже покрывает — НО нужно: (а) реально завести профили и вписать `url`; (б) тянуть число/звёзды через API площадок (не хардкодить «5.0» — правило G-3); (в) кнопка «Оцените нас» ведёт на нужную площадку.

**D. Честный микс +/-.** Публиковать не только 5/5. Показать 1–2 реальных «с замечанием» + ответ компании («приняли, исправили»). Это поднимает доверие ко всем остальным (Forbes 2024). `Review.approved` (модерация) позволяет фильтровать, но НЕ прятать негатив полностью.

**E. Отвечать на отзывы.** Добавить поле `reply?: { author; text; date }` в `Review`. Ответ на внешней площадке + зеркало на `/reviews`. Сигнал «живая компания».

**F. Дисклеймер «реальные отзывы».** На `/reviews` и под `RatingBadge` — микро-текст: «Отзывы с реальных мероприятий, подтверждённые фото/видео события. Внешний рейтинг — на Яндекс.Картах / 2ГИС / Google.» Не «★★★★★ супер».

## ИТОГ ДЛЯ ПОЛЬЗОВАТЕЛЯ

Отзывам **сейчас не верят (2.5/10)**, потому что на сайте как описано — 3 сочинённых цитаты, пустой бейдж внешнего рейтинга и ноль объёма. Инструменты доверия в спеке **хорошие (6/10)**: `RatingBadge` на Яндекс/2ГИС/Google, видео-отзывы, `EventsRecap`, `ReviewCard` — но **нет самой трубы сбора**. Главный gap (RT-1/RT-2): нет авто-запроса после события и нет CMS-конвейера отзывов (это же баг из `29_CMS_CRITIC`). Закрыть RT-1…RT-4 → вера поднимется до ≥8/10: реальный объём (≥20) + видео + внеш. площадки с живым рейтингом + честный микс +/- + ответы компании.
