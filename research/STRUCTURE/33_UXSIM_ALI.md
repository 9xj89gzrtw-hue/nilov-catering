# 33 — UX-СИМУЛЯЦИЯ: Али (халяль-клиент)

**Роль:** Халяль-клиент Али (персонаж #18 из `32_UX_PERSONAS`).
**Сценарий:** «Нужен халяль-кейтеринг. Хочу найти халяль-сертификат и халяль-меню, понятно ли это заявлено».
**Тон:** критик-персонаж, занижение. Балл ожидаемо ≤8.
**Источники:** `04_BLOCKS.md` (БЛОК 16 MenuCatalog, БЛОК 17 DishCard), `02_IA.md` (роут `/certificates`), `23_MENU_STRATEGY.md` (§3 «Халяль ≥6 блюд», §3.1 честный статус), `32_UX_PERSONAS.md`.

---

## ПРОИГРЫШ СЦЕНАРИЯ (заход → халяль → сертификат)

1. **Заход (главная `/`).** В Hero-microcopy — «Маркировка по 14 аллергенам», **но про халяль — НИ СЛОВА**. В nav (События/Меню/Галерея/Почему мы/Команда/Отзывы/Сезонное) **нет халяль-входа**. Али не видит халяль как опцию с порога — он для сайта «невидим» на главной. → единственный путь — «Смотреть меню» → `/menu`.

2. **Меню (`/menu` → `/menu/catalog`).** На ленде `/menu` халяль **не заявлен** (там форматы/подборки). В **каталоге** (`/menu/catalog`) есть фильтр-чип `halal` (БЛОК 16) — это плюс: Али может отсечь одним тапом. Чип ведёт на подборку «халяль ≥6 блюд» (`23` §3).

3. **Халяль-блюда (DishCard, БЛОК 17).** Здесь первая проблема честности: бейдж халяля рендерится по `halalStatus` (`23` §3.1 / B4):
   - `'certified'` → зелёный «✓ Халяль» — **но по статусу линии халяль = «ПО ЗАПРОСУ, ПРИ НАЛИЧИИ СЕРТИФИКАЦИИ»**, то есть `certified` сейчас **не наступил** (сертификация не подтверждена заказчиком).
   - `'on-request'` → нейтральный «Халяль по запросу» — это то, что Али **реально увидит**. То есть сайт говорит: «мы можем сделать халяль, но сертификата НЕТ, пишите менеджеру». Для клиента, чья вера требует **именно сертифицированный** халяль — это провал обещания ещё до сертификата.

4. **Сертификат (главная боль Али).** Али ищет **халяль-сертификат**. Роут `/certificates` (`02_IA`) существует, но там — **152-ФЗ, ТР ТС 021/2011, Декларация соответствия, Роспотребнадзор, аккредитация кухни**. Это пищевая безопасность, **НЕ религиозная халяль-сертификация**. Никакой исламской сертификации (Халяль-центр/Стандарт РФ 56085 и т.п.) на сайте **нет и не предполагается**. → Али заходит на `/certificates`, не находит халяль-сертификат, видит «пишите менеджеру». **Тупик.**

5. **Связность.** Hero/микрокопи не ссылаются на халяль. На `/events/*` халяль не упомянут. Нигде нет CTA «Халяль-кейтеринг» с deep-link на `?diet=halal`. Разрыв логики: опция есть в каталоге, но «не заявлена» нигде на входе.

## ОЦЕНКА (0–10 по 4 осям, занижение)

| Ось (`32_UX_PERSONAS`) | Балл | Почему |
|---|---|---|
| 1. Понятность структуры | **5** | Путь до халяль-меню логичен (`/menu → /menu/catalog → чип halal`), но халяль **не заявлен** на главной/в nav/на `/menu`-ленде. «Сирота»: опция есть, входа нет. |
| 2. Удобство действий | **4** | Найти халяль-меню — 3 тапа, без deep-link из Hero. Найти **сертификат** — **невозможно**: `/certificates` про другое, исламской сертификации на сайте нет. Главная цель Али (сертификат) — тупик. |
| 3. Дискомфорт/перегрузка | **3** | Честный статус «по запросу» — правильно по `B4`, НО он **прячет отсутствие сертификата**: Али думает «есть халяль», а по факту «сертификата нет, пишите менеджеру». Смешанный сигнал + скрытый разрыв. |
| 4. Устройства (mobile/tablet/desktop) | **5** | Чип `halal` доступен на mobile в фильтр-баре (`БЛОК 16` scroll-snap). Но заявления халяля на входе нет нигде; sticky-CTA «Спланировать» к сертификату не ведёт. Mobile-путь к сертификату = тот же тупик. |

**ИТОГ:** (5+4+3+5)/4 = **4.25 → 4/10**. (Ожидание ≤8 подтверждено; далеко от цели >9.)

## BUG-LIST (что исправить для ≥9)

> Приоритет: сначала «видимость опции» и «сертификат» (главные боли Али), затем честность подачи.

- **B-ALI-1 (критично).** Заявить халяль **на входе**: добавить на главную (Hero-microcopy или TrustBar-бейдж) и в nav/на `/menu`-ленд строку вида «Халяль — по запросу, при наличии сертификации» + deep-link `→ /menu/catalog?diet=halal`. Сейчас опция «невидима» с порога.
- **B-ALI-2 (критично, тупик).** Либо (а) **добавить реальную халяль-сертификацию** и показать скан/бейдж на `/certificates` (исламский сертификат, не только 152-ФЗ), либо (б) **честно развести**: создать на `/certificates` (или в FAQ) явный блок «Халяль» с прямым ответом «сертифицированная линия — по запросу, пишите менеджеру, вот контакт». Сейчас Али идёт на `/certificates` и не находит халяль вообще.
- **B-ALI-3.** На DishCard бейдж `halalStatus:'on-request'` должен **нести ссылку/CTA «узнать про сертификат»** (ведёт на тот же блок B-ALI-2), а не быть «висячим» нейтральным ярлыком. Иначе Али видит «по запросу» и не знает, куда писать.
- **B-ALI-4.** Hero-microcopy «Маркировка по 14 аллергенам» — **добавить** рядом явное упоминание диет-опций (веган/БГ/**халяль по запросу**) со ссылкой на каталог-фильтр. Сейчас халяль в обещаниях главной отсутствует.
- **B-ALI-5.** На страницах `/events/*` (особ. корпоратив/частное, где халяль востребован) добавить микро-упоминание «возможен халяль-формат по запросу» + ссылку. Разрыв: халяль есть в меню, но не на событийных посадочных.
- **B-ALI-6.** Фильтр `halal` в MenuCatalog: при статусе `on-request` показывать **предупреждение-баннер** вверху подборки («Это блюда готовятся как халяль ПО ЗАПРОСУ, без постоянного сертификата — подтвердите у менеджера»), чтобы Али не принял «по запросу» за «сертифицировано».
- **B-ALI-7.** Дедуп с `B4` (`23` §3.1): зафиксировать, что `halalStatus:'certified'` **не рендерится**, пока заказчик не подтвердит сертификацию. До того — сайт честно НЕ обещает сертификат, но ОБЯЗАН дать Али понятный ответ «где сертификат / как заказать халяль» (см. B-ALI-2).

## ВЕРДИКТ (коротко)
Халяль как опция **существует** (фильтр-чип в каталоге, бейдж на карточке) — но **не заявлен** ни на главной, ни в nav, ни на `/menu`-ленде: клиент «не видит» опцию с порога. Главная боль Али — **халяль-сертификат** — **не закрыта**: роут `/certificates` про пищевую безопасность (152-ФЗ/ТР ТС), исламской сертификации на сайте нет, а честный статус «по запросу» лишь **прячет отсутствие сертификата**, не давая ответа. Для верующего клиента, которому нужен именно сертифицированный халяль, сайт **проваливает сценарий**. Балл **4/10**.
