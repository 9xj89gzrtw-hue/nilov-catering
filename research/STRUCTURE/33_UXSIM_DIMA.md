# 33 — UX-СИМУЛЯЦИЯ: Дима (только телефон, метро, медленный 4G)

**Роль:** критик-персонаж №25 из `32_UX_PERSONAS` — технический сценарий «мобильный пользователь». НЕ автор. Настроен на занижение.
**Сценарий:** «У меня только телефон. Еду в метро, 4G слабый и нестабильный. Хочу всё делать в 1 тап и чтобы ничего тяжёлого не грузилось и не тормозило. Если надо ждать загрузку или скроллить лишнее — ухожу.»
**Устройство:** mobile (iPhone SE / бюджетный Android, ≤4 cores, 4G в туннеле). Фокус: mobile-производительность (прелоадер, LCP, вес страницы), тач-цели ≥44px, «всё в 1 тап», нет ли тяжёлых блоков на слабом соединении.
**Девиз:** «Покажи контент мгновенно и дай нажать 1 раз. Не грузи видео и не заставляй ждать.»

## ПРОИГРЫШ СЦЕНАРИЯ (от первого лица)

**Заход с телефона (главная `/`):**
- Открываю сайт в метро. Первое, что вижу — **блокирующий прелоадер** (`04` блок B5): `position:fixed; inset:0`, держит экран **1.4–1.9s** (+ exit-анимация) без учёта `prefers-reduced-motion`. 🔴 На 4G это ≈ вечность: я НЕ могу ни скроллить, ни тапнуть, пока лого «морфится» в Hero. INP убит сразу. (1-й красный флаг — сайт не отвечает на входе.)
- Когда прелоадер уходит, Hero: H1 + CTA «Спланировать событие» → `/plan`. Под CTA — цена-строка (если применён волновой фикс). Ниже — `TrustBar` (marquee логотипов, CSS-анимация), `EventTypeSelector`, `FormatShowcase`.
- 🔴 **Hero-video (Б6):** на «слабом 4G в метро» `effectiveType` часто = `'4g'` (браузер так помечает), а iPhone SE имеет cores>4 → квота НЕ срабатывает → **видео ГРУЗИТСЯ и autoplay-ится** поверх poster. На 4G это бьёт LCP/INP: первый осмысленный кадр прилетает с задержкой, скролл дёргается. Poster как `<img fetchpriority=high>` — единственный LCP-кандидат — перекрыт загрузкой видео.
- 🔴 **Lenis (Б7):** `smoothTouch:false` в Фазе 0 — вопрос открыт. Если НЕ выключен (`touchMultiplier:2`), скролл «тягучий», враждебен нативной инерции телефона → раздражает и грузит INP при каждом свайпе.

**Сколько тапов до чего-либо (аудит «1 тап»):**
- *Заявка / расчёт:* `StickyMobileCTA` снизу (1 тап) → но ведёт в `/plan` (хаб) или `/plan/calculator` — 1 тап до инструмента ✅. Однако дальше калькулятор = формат+гости+тариф (3–4 шага). Для «просто заказать» — многовато.
- *Галерея / Отзывы:* `MobileBottomNav` = ровно 5 пунктов (События/Меню/Галерея/Почему мы/Спланировать). **Отзывы (`/reviews`) и Команда/Сезонное — НЕ в bottom-nav** → только через бургер-меню → **2–3 тапа** до отзывов. Для скептика/обычного юзера это не «1 тап».
- *Меню:* bottom-nav «Меню» → `/menu` → выбор формата → `/menu/{format}`. **2–3 тапа** до конкретного меню. Экспресс-deep-link («Кофе-брейк на 15») есть, но только для свадьбы/кофе-брейка — для остальных сценариев 1-тапа нет.

**Тяжёлые блоки ниже фолда (грузятся на 4G «бесшумно»):**
- 🔴 `HomeVideoShowcase` (Блок 6-bis) — автоplay по IntersectionObserver (`threshold:0.5`), источник Rutube/Vimeo facade. На 4G iframe-эмбед грузится при первом появлении в вьюпорте → тратит трафик/полосу, хотя я его ещё не доскроллил.
- 🔴 `EventsRecap`, `TestimonialsCarousel` (video-wall), `GalleryMasonry` (20–30% плиток = видео), `InspireStrip` (1 клип). Каждый потенциально тянет видео на слабом соединении.
- 🔴 Framer Motion везде (clip-path wipe H1, Ken Burns на карточках, parallax `MenuWowGesture`, marquee TrustBar) + `MenuWowGesture` (fullscreen parallax-карусель) — JS/CSS-вес и layout-thrash на бюджетном телефоне → INP растёт, скролл не плавный.

**Тач-цели и навигация (Б12/Б3/Б4):**
- Global-правило `a,button,input{min-height:44px}` есть, но 🔴 **gap между пунктами `MobileBottomNav` = `gap-1` (<8px, Б12)** — соседние тап-зоны вплотную, случайные тапы при тряске в метро.
- 🔴 OR-правило `StickyMobileCTA` ⇄ `MobileBottomNav`: если оба видимы одновременно — два sticky снизу перекрывают контент и дублируют кнопку «Спланировать». Нужна строгая проверка на real-mobile.
- 🔴 Наложение оверлеев: `AnnouncementBar`(z-110) + `CookieBanner`(z-120) + `StickyMobileCTA`(z-90) на маленьком экране SE могут **заблокировать клик** по контенту, пока куки-баннер не закрыт. Закрытие — только крестик `×` (Б12 требует ≥44px «Понятно»).

## BUG-LIST (что исправить для ≥9)

**Критичные (блокируют «мгновенно + 1 тап» на 4G):**
1. **Блокирующий прелоадер (Б5).** 1.4–1.9s `position:fixed inset:0` убивает LCP/INP на 4G — Дима ждёт и не может тапнуть. → Убрать ИЛИ morph ≤400ms без блокировки paint (`useReducedMotion` → мгновенно `opacity:1`). ЖЁСТКО: на mobile прелоадер НИКОГДА не держит `fixed` дольше 400ms.
2. **Hero-video на слабом 4G (Б6).** Квота `effectiveType<='4g'` трактует метро-4G как «показать только poster», но на iPhone SE (cores>4) она НЕ срабатывает → видео грузится. → На mobile агрессивнее: ТОЛЬКО poster `<img fetchpriority=high>`, видео `preload="none"`, строго по клику/Wi-Fi. Никакого autoplay на сотовом.
3. **Lenis на тач (Б7).** Если `smoothTouch` не выключен — тягучий скролл на mobile. → Фаза 0: `smoothTouch:false` ИЛИ отключать Lenis при `@media (pointer:coarse)`. Нативная инерция телефона — лучше.
4. **Видео ниже фолда грузятся на 4G.** `HomeVideoShowcase`/`EventsRecap`/`TestimonialsCarousel`/`GalleryMasonry` (20–30% видео) тянут iframe/MP4 при появлении во вьюпорте. → На слабом соединении НЕ autoplay; facade + play-кнопка, iframe грузится только ПО КЛИКУ. Экономит трафик/полосу Димы.

**Важные (mobile-комфорт / 1 тап):**
5. **«1 тап» не универсален.** Отзывы/Команда/Сезонное/Меню требуют 2–3 тапов (бургер/хаб). → Поднять приоритетные чипы (Отзывы уже в бургере по волне 4) + дать deep-link express-пресеты не только для свадьбы/кофе-брейка, но и для корпоратива/детского/выпускного прямо на главной/Hero.
6. **Тач-цели и gap (Б12).** `gap-1` между пунктами `MobileBottomNav` <8px → случайные тапы в трясущемся метро. → Увеличить gap ≥8px; проверить высоту ВСЕХ CTA (в т.ч. «?»-тултипы, крестики модалок, чипы фильтров) на ≥44px.
7. **Оверлеи блокируют клик (Б4/Виктор V7).** `AnnouncementBar`+`CookieBanner`+`StickyMobileCTA` на маленьком экране перекрывают контент. → Куки-баннер НЕСЁТ крупную (≥44px) кнопку «Понятно» (НЕ только ×); свести число одновременных оверлеев к минимуму; strict OR-правило sticky (ровно 1 снизу).
8. **Вес страницы / Framer везде.** clip-path wipe + Ken Burns + parallax `MenuWowGesture` + marquee на бюджетном телефоне → INP растёт. → `prefers-reduced-motion` → плоский рендер (без Ken Burns/parallax/wipe); lazy-load тяжёлых жестов; бюджет LCP <1.2s / INP <200ms верифицировать Lighthouse на throttled 4G.

**Точность данных:**
9. **Цены `pending-verification`** (Фуршет+Кофе-брейк «от 390 ₽» на живом сайте vs канон 2 450 / 950) — если Дима увидит завышение, недоверие. → Сверить `PricingConfig` с прайсом заказчика ДО прод-релиза (BUG-F2).

## ВЕРДИКТ
Сайт **структурно понятен**, но для персонажа «только телефон, метро, медленный 4G» две ключевые боли не закрыты: **(1) контент НЕ мгновенный** (блокирующий прелоадер 1.4–1.9s + hero-video/видео ниже фолда грузятся на 4G, Lenis тянет скролл) и **(2) «1 тап» работает только для заявки/расчёта**, а Галерея/Отзывы/Меню/Команда требуют 2–3 тапов. Плюс риски наложения тач-зон и оверлеев на маленьком экране.
Исправить #1–#4 (прелоадер/видео/Lenis/оверлеи) + #5–#6 (1 тап/тач-цели) → балл поднимается до 8–8.5. До ≥9 нужна верификация Lighthouse на throttled 4G + real-mobile тест навигации.
