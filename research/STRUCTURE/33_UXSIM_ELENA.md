## ПРОИГРЫШ СЦЕНАРИЯ (от первого лица)

**1. Заход на `/` (главная)**
- Hero: «Кейтеринг, который чувствуешь заранее» + CTA «Спланировать событие» → `/plan` и «Смотреть меню» → `/menu`. Цены за гостя на 1-м экране — есть микро-строка («Свадьба/банкет от 4 470 ₽…»), но про «юбилей 120 чел» ни слова.
- `EventTypeSelector` (блок 4): 5 карточек — Свадьба / Детский / Корпоратив / Частное / Chef-at-home. **«Юбилей» как типа НЕТ.** Я — секретарь, организую «юбилей фирмы». Догадываюсь, что это «Корпоратив». Жму → `/events/korporativ`.
  → Скрытый разрыв: в галерее (`GalleryTeaser`) капшены говорят «Юбилей», в IA — только «Корпоратив». Язык не совпадает с моим запросом.

**2. /events/korporativ (контекст события)**
- Страница даёт контекст юбилея → «рекомендованные форматы из /menu». Т.е. ссылки на `/menu/furshet`, `/menu/banquet`. **Никакой сравнительной таблицы «банкет vs фуршет» на месте нет.**
- Чтобы сравнить, мне надо ОТКРЫТЬ ДВЕ разные страницы меню и держать их в голове. Для не-спеца это когнитивная нагрузка. Где «для 120 человек что выгоднее/удобнее» — не сказано.

**3. Сравнение банкет/фуршет**
- `FormatShowcase` (блок 5, на главной ниже фолда): карточки Фуршет «от 2 450 ₽/гость» (бейдж «Хит корпоративов»), Банкет «от 4 470 ₽/гость» (бейдж «Для свадеб»), Кофе-брейк «от 950 ₽». Подзаголовки понятные («Лёгкие закуски на стоячем приёме» / «Посадка, официанты, классика»). Это ПЛЮС — язык адаптирован.
- НО: это только «от X ₽» и одна фраза. **Нет таблицы:** что входит, сколько официантов, посадка vs стоячий приём, для 120 чел что реально. Чтобы понять разницу для юбилея, иду в `/menu/furshet` и `/menu/banquet` по отдельности. Сравнения «в одном окне» нет.
- Банкет-карточка помечена «Для свадеб» — для юбилея 120 это сбивает (я же не невеста). Язык подсказки не про мой случай.

**4. Получение КП (коммерческого предложения)**
- Самый болезненный момент. «Получить КП быстро» — как? Пути:
  - (а) `/plan/calculator`: формат → гости (120) → тариф. Получаю цену. **Но КП как документ (PDF/предложение для босса) НЕ генерится** — в `08_CONSTRUCTOR_SPEC` описан только расчёт + шаг 6 «Контакт» (submit → `/api/contact`). Про «Скачать КП» ни слова. `02_IA` упоминает «`/offer` PDF-КП генерится в `/plan/calculator`», но в спеце `08` это не реализовано.
  - (б) Конструктор 6 шагов — для секретаря, которой нужно «сравнить и выдать КП», это ПЕРЕГРУЗ (формат → гости → тариф/своё → допуслуги → итог → контакт).
  - (в) Итог КП = заявка менеджеру (оставить имя/телефон/дату), менеджер перезвонит. **НЕ self-service «быстро».** Скрытый обязательный шаг: чтобы получить предложение, надо оставить контакты.
- Для 120 гостей рекомендация «50+ → Банкет» (B-MUX-5) появляется только ВНУТРИ конструктора на шаге 2. Я её не вижу, пока не начну собирать меню. А мне нужно решение ДО входа в тяжёлый инструмент.

**5. Терминология**
- Конструктор (шаг 1) предлагает 5–6 форматов, включая «Мобильный фуршет» — для секретаря лишняя, путающая опция (что это вообще?). ФорматShowcase её не показывает, а конструктор — да. Неконсистентность.

## BUG-LIST (что исправить для ≥9)

**Критичные (блокируют сценарий «сравнить + КП»):**
1. **Нет инструмента сравнения форматов.** Чтобы сравнить банкет/фуршет, Елена открывает `/menu/furshet` и `/menu/banquet` по отдельности. → Сделать блок/страницу «Банкет или фуршет?»: таблица (цена/гость, что входит, посадка vs стоячий приём, официанты, для 120 чел) прямо на `/events/korporativ` и/или в `FormatShowcase`. Один экран, не два.
2. **«Получить КП быстро» ≠ self-service.** Калькулятор даёт цену, но PDF-КП не генерится (`02_IA` обещает `/offer` в калькуляторе, `08` этого не реализует). → Добавить кнопку «Скачать КП (PDF)» в `/plan/calculator` с данными заявителя. Иначе КП = только звонок менеджера, не «быстро».
3. **Скрытый обязательный шаг для КП:** чтобы получить предложение, надо оставить имя/телефон/дату (шаг 6 конструктора / контакт-форма). → Сделать предпросмотр КП без обязательных контактов + опц. «оставить контакт для финального КП». Секретарь хочет «прикинуть и показать боссу» без барьера.

**Важные (ясность для не-спеца):**
4. **«Юбилей» не первый-class путь.** Спрятан под «Корпоратив»; в галерее — «Юбилей». → Добавить алиас/ярлык «Юбилей компании → Корпоратив» на `EventTypeSelector` и `/events/korporativ`, чтобы секретарь узнала свой кейс.
5. **Рекомендация формата появляется слишком поздно** (B-MUX-5 «50+ → Банкет» только внутри конструктора, шаг 2). → Показать подсказку «Для 120 гостей рекомендуем Банкет» ДО входа в инструмент (на `/events/korporativ` и в `FormatShowcase`/сравнении).
6. **Банкет-карточка помечена «Для свадеб»** — сбивает при юбилее. → Сделать бейдж контекстным («Для посадки 50+») или добавить «Юбилей/корпоратив» как вариант.
7. **«Мобильный фуршет» в конструкторе путает не-спеца** (нет в `FormatShowcase`). → Убрать из быстрого пути или объяснить plain-языком; консистентность с `FormatShowcase`.
8. **6-шаговый конструктор тяжёл для «сравнить + КП».** → Дать быстрый пресет «Корпоративный юбилей на 120» (формат + пример меню + итог + кнопка КП) за 1–2 тапа, как для Игоря/Ольги.

**Точность данных:**
9. **Цены `pending-verification` расходятся с живым сайтом** (Фуршет по спеке «от 2 450 ₽», на сайте «от 390 ₽»; Банкет «от 4 470 ₽»). КП с неверной ценой = потеря доверия босса. → Сверить `PricingConfig` с актуальным прайсом (BUG-F2) ДО релиза.

## ВЕРДИКТ
Сайт **логичен по структуре**, но для персонажа «секретарь, юбилей 120 чел» две ключевые цели не закрыты: **(1) нет единой точки сравнения банкет/фуршет** (разбросано по страницам меню) и **(2) нет мгновенного self-service КП** (только заявка менеджеру с обязательным вводом контактов). Плюс терминология и рекомендации не адаптированы под не-спеца, а цены в расчёте рискуют быть неверными. Исправить #1–#3 (+ #9) → балл поднимается до 8.5–9.
