# 33 — UX-симуляция: Веган Лиза (перезапуск)

**Роль:** #17 «Веган Лиза» — «найти веган-линию, не искать по всему меню».
**Тон:** критик-персонаж, настроен на занижение. Средний балл по всем 30 персонажам должен быть >9 — Лиза его тянет вниз.
**Сценарий:** «Ищу веган-линию для мероприятия. Хочу найти веганские блюда НЕ по всему меню, понятно ли их выделили».
**Проход:** заход → веган-меню → фильтр. Ищем: есть ли веган как отдельная линия, понятен ли фильтр, не спрятан ли.

---

## Проигрыш сценария (шаг за шагом)

1. **Заход на `/` (главная).** Hero → TrustBar → EventTypeSelector (6 типов событий) → FormatShowcase (фуршет/банкет/кофе-брейк) → MenuPreview («Фуршет на 50» / «Банкет под ключ» / «Кофе-брейк для офиса»). **Веган-линии / диетического входа НЕТ ни в одном блоке.** Лиза с порога не видит, куда идти за веганом.
2. **Навигация «Меню».** В шапке `Меню` → `/menu`. Лендинг `/menu` ведёт в форматы: `/menu/furshet`, `/menu/banquet`, `/menu/coffee-break`, `/menu/detskoe`, `/menu/catalog`, `/menu/show-cooking`. **Отдельного `/menu/vegan` (или `/menu/vegan-line`) НЕТ.** Веган — не формат, поэтому он не попал в эту сетку роутов (`02_IA`).
3. **Поиск вегана.** Единственный путь — `/menu/catalog` (поштучный каталог с фильтрами категория/диета/цена) → включить фильтр `диета = vegan`. Либо в конструкторе (`08`, шаг 3) «линия по станциям + фильтр диета». **Оба пути = фильтр внутри каталога, а не выделенная линия-страница.**
4. **Просмотр по формату (ловушка).** Если Лиза заходит в `/menu/furshet` или `/menu/banquet`, веган-блюда (`dietBadges:['vegan']`) перемешаны с мясными. Чтобы собрать «только веган», надо листать всё меню или уходить в каталог+фильтр. **Требование «НЕ по всему меню» нарушено** при форматном просмотре.
5. **Бейдж vs линия.** На карточке `dietBadges` виден (по `04`/`23`). Но это пометка на блюде, а не навигационная единица. Лиза видит слово «веган» только после того, как сама отфильтровала каталог.

---

## Что НАШЛИ (плюсы, чтобы быть честным)

- `dietBadges: ('vegan'|'gluten-free'|'halal')[]` задан в модели `Dish` (`04_BLOCKS`, CMS-коллекция) и виден на `DishCard` — веган помечается на карточке.
- Веган = **счётная линия ≥9 блюд** (`23` §3): будда-боул с тофу, стейк из цветной капусты, карпаччо из свёклы, овощное рагу, шоколадный мусс (авокадо), веган-граzing, мини-тако с цветной капустой, цуккини-пюре с кокосовым карри, веррин чиа-пудинг. Данные реально есть.
- В `/menu/catalog` и в конструкторе (`08`) есть фильтр по диете → веган отсеять технически можно.
- `AllergenFilter` (`07`) + per-dish exclude (`08`) — диеты/аллергены учтены в модели данных.

## Что НЕТ (минусы, глазами Лизы)

- **Нет выделенной веган-линии как роута.** Форматная IA (`/menu/{format}`) не покрывает диеты. Веган спрятан за каталогом+фильтром.
- **Несимметричность с `/menu/detskoe`.** Детское получило отдельный роут (`02_IA`, Волна 1 B-OLGA-2), веган/БГ/халяль — нет, хотя в `23` §3 все они равноправные «счётные линии». Почему детское — страница, а веган — только бейдж? Логика разорвана.
- **Нет видимого CTA «Веган / Диет-линии»** на `/menu` и на главной. Лиза не знает, что фильтр существует, пока не дойдёт до каталога. Discovery = ноль.
- **Форматный просмотр заставляет листать мясо** — прямо против сценария «НЕ по всему меню».
- **Фильтр не «умный»**: после выбора vegan нет подсказки «собрать веган-мероприятие» / перехода в конструктор с пред-фильтром. Разрыв между просмотром и сборкой (`08`).

---

## Оценка по 4 осям (занижение, 0–10)

| Ось | Балл | Почему |
|---|---|---|
| 1. Понятность структуры | **4** | Меню разбито по форматам, веган не выделен в линию; противоречие с `/menu/detskoe`. IA нелогична для диет-запроса. |
| 2. Удобство действий | **4** | Путь: Главная → Меню → Каталог → фильтр диета. 3+ клика, вход в веган неочевиден, discovery слабый. |
| 3. Дискомфорт/перегрузка | **4** | На форматных страницах веган утоплен в мясных; «не по всему меню» не выполняется; каталог 78 SKU давит. |
| 4. Устройства (mobile) | **5** | Bottom-nav «Меню» есть; чип-фильтр работает на mobile, НО только если дойдёт до каталога — а вход спрятан за форматами. |

**Средний балл Лизы: ≈ 4.25 / 10** (ожидаемо ≤8, тянет общий средний вниз — цель >9 недостижима, пока диет-линии не вынесены в роуты).

## Bug-list (что исправить, чтобы Лиза дала ≥9)

- **B-LIZA-1 (блокирующий для диет-персон):** Добавить выделенную диет-линию как роут(ы): `/menu/vegan` (+ опц. `/menu/gluten-free`, `/menu/halal` по тому же паттерну, что `/menu/detskoe`). Линия = пред-отфильтрованный `MenuCatalog` по `dietBadges`. Закрывает «не по всему меню».
- **B-LIZA-2:** На ленде `/menu` добавить блок/чипы «Диет-линии» рядом с форматами: Веган · Безглютен · Халяль · Детское — симметрично `/menu/detskoe`. Убрать несоответствие.
- **B-LIZA-3:** На главной `MenuPreview` (`04`, блок 6) добавить коллекцию «Веган-линия» (или общий вход «Диетические меню») с CTA → `/menu/vegan`.
- **B-LIZA-4:** Сделать фильтр `диета` **всегда видимым чип-баром сверху** `/menu/catalog` (НЕ в выпадающем меню), с `vegan` на первом месте. Закрывает «понятен ли фильтр, не спрятан ли».
- **B-LIZA-5:** При выборе `vegan`-фильтра в каталоге показывать CTA «Собрать веган-мероприятие → /plan/constructor?diet=vegan» (пред-фильтр State в `08`). Связать просмотр и сборку, убрать разрыв.
- **B-LIZA-6:** На форматных страницах (`/menu/furshet` и т.д.) добавить переключатель «Только веган» (quick-filter по `dietBadges`), чтобы Лиза не листала мясо.
- **B-LIZA-7 (a11y):** `dietBadges` на `DishCard` — убедиться, что бейдж веган имеет `aria-label` и цветовой контраст AA (не только зелёный цвет), иначе скринридер/цветослепые не различат.
