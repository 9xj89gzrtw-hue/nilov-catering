# 33 — UX-СИМУЛЯЦИЯ: Нина (аллергия на глютен)

**Роль:** Клиент с аллергией на глютен (персонаж #19 из `32_UX_PERSONAS`).
**Сценарий:** «Аллергия на глютен. Хочу найти БЖУ/аллергены по КАЖДОМУ блюду — понятно ли это».
**Тон:** критик-персонаж, занижение. Балл ожидаемо ≤8.
**Источники:** `04_BLOCKS.md` (БЛОК 16 MenuCatalog, БЛОК 17 DishCard), `02_IA.md`, `23_MENU_STRATEGY.md` (§0), `32_UX_PERSONAS.md`.

---

## ПРОИГРЫШ СЦЕНАРИЯ (заход → меню → аллергены по блюду)

1. **Заход (главная `/`).** В Hero-microcopy есть обещание «Маркировка по 14 аллергенам» — но это просто текст, **без ссылки** на каталог/фильтр. Нина не знает, куда идти за аллергенами. → клик «Смотреть меню» → `/menu`.
2. **Меню (`/menu` → `/menu/catalog`).** В каталоге **ЕСТЬ фильтр-чип `gluten-free`** (БЛОК 16) — это плюс: Нина может отсечь всё с глютеном одним тапом. Фильтр ведёт на реальную подборку (БГ ≥11 блюд по `23` §3).
3. **Аллергены по блюду (DishCard, БЛОК 17).** Здесь спотыкаемся:
   - `gluten-free` **диет-бейдж виден сразу** на лице карточки — для БГ-блюд вердикт ясен («безопасно»).
   - Но **14 аллергенов-чипов + КБЖУ НЕТ на лице карточки** — они «раскрываются на hover/фокус» (БЛОК 17: «Раскрываются на hover/фокус … И доступны клавиатурно»; БЛОК 16 hover-detail). То есть, чтобы увидеть, **есть ли в блюде глютен** (для НЕ-GF позиций) — надо навести курсор (desktop) или **тапнуть развернуть** (mobile).
   - `crossContact` ⚠ («перекрёстная контаминация») — тоже внутри этого hover-группа. Для тяжёлой аллергии это самое важное предупреждение — и оно **спрятано за hover/tap**.

4. **Вердикт «содержит глютен / не содержит»:** у БГ-блюд — понятен (бейдж). У остальных — **непонятен сразу**: нет явного глифа «содержит глютен» на лице карточки, только отсутствие бейджа GF. Чтобы убедиться — hover/tap.

5. **Реальность данных (критично).** По `23` §0 / `25_MENUCRITIC_FINAL` R1 (БЛОКИРУЮЩИЙ): поля заданы, но **значения 14 аллергенов — `pending`, не заполнены ни для одного из 78 блюд**. То есть даже если Нина раскроет карточку — «маркировка по 14 аллергенам» сейчас **пустая**. Обещание Hero — декларация контракта, не факт.

## ОЦЕНКА (0–10 по 4 осям, занижение)

| Ось (`32_UX_PERSONAS`) | Балл | Почему |
|---|---|---|
| 1. Понятность структуры | **7** | Путь `/menu → /menu/catalog` логичен, фильтр `gluten-free` есть и ведёт на реальную подборку. Минус: Hero-обещание аллергенов не слинковано, нет выделенной страницы/легенды аллергенов. |
| 2. Удобство действий | **5** | Фильтр БГ — удобно. Но **БЖУ и чипы 14 аллергенов требуют hover/tap на КАЖДОЙ карточке**; глютен-глиф не на лице. Проверить конкретное блюдо = лишнее действие на каждое. |
| 3. Дискомфорт/перегрузка | **4** | Аллергены спрятаны за hover/tap — это ровно то, что инструкция персонажа велит штрафовать («надо куда-то отдельно нажать»). `crossContact` ⚠ тоже hover-only = риск для здоровья. R1 = данных нет вообще. |
| 4. Устройства (mobile/tablet/desktop) | **5** | На mobile hover-detail «заменён на tap-to-expand» (БЛОК 17) — Нине надо тапать каждую из 78 карточек, чтобы увидеть аллергены/КБЖУ. Sticky-CTA «Спланировать» не помогает с аллергенами. |

**ИТОГ:** (7+5+4+5)/4 = **5.25 → 5/10**. (Ожидание ≤8 подтверждено; далеко от цели >9.)

## BUG-LIST (что исправить для ≥9)

> Приоритет по влиянию на безопасность/удобство аллергичного клиента.

- **B-NINA-1 (критично, блокирующий опыт).** Заполнить значения 14 аллергенов до прода (`23` §0 R1). Сейчас `pending` для всех 78 блюд — «маркировка по 14 аллергенам» пустая, Нина не может доверять ни одному чипу. *Без этого всё остальное бессмысленно.*
- **B-NINA-2.** Глютен-статус должен быть **всегда виден на лице DishCard**, не за hover/tap. Минимум: явный глиф «🌾 содержит глютен» у НЕ-GF блюд (сейчас только отсутствие бейджа GF = неочевидно). БЖУ (ккал + белки) — хотя бы кратко на лице, полное КБЖУ при раскрытии.
- **B-NINA-3.** `crossContact` ⚠ («перекрёстная контаминация») — **всегда видимый**, не часть hover-группы. Для тяжёлой аллергии скрытое предупреждение = риск.
- **B-NINA-4.** На **mobile** аллергены/КБЖУ не прятать за tap на каждой карточке — показывать компактной строкой под фото по умолчанию (tap-to-expand только для ПОЛНЫХ КБЖУ). 78 карточек × tap недопустимо.
- **B-NINA-5.** Легенда 14 аллергенов ТР ТС 021/2011 (что значат чипы, включая глютен) — модалка или страница `/allergens` + ссылка из каталога. Для неспециалиста аббревиатуры непрозрачны.
- **B-NINA-6.** Связать Hero-microcopy «Маркировка по 14 аллергенам» с **прямой ссылкой** на `/menu/catalog?diet=gluten-free` (иначе пустое обещание, разрыв логики).
- **B-NINA-7.** Фильтр `gluten-free` — добавить режим «показать всё, пометив глютен-содержащие» (не только прятать не-GF), т.к. Нина часто хочет видеть меню целиком с пометками, а не только узкую БГ-подборку.
- **B-NINA-8.** Дедуп-вердикт: на лице карточки БГ-блюд бейдж `gluten-free` + у НЕ-GF — глиф «содержит глютен» должны быть **одинаково заметны** (сейчас бейдж приоритетен, а «содержит» — скрыт). Симметрия важна для сканирования взглядом.

## ВЕРДИКТ (коротко)
Структура и фильтр БГ — на уровне. Но **аллергены + БЖУ спрятаны за hover/tap**, `crossContact` скрыт, а сами значения аллергенов ещё **не заполнены** (R1). Для клиента с аллергией, чья главная задача — «увидеть глютен сразу», сайт **не проходит**: нет мгновенного, честного вердикта по каждому блюду. Балл **5/10**.
