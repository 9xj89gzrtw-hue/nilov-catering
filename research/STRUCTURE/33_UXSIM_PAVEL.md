# 33 — UX-СИМУЛЯЦИЯ: Гендиректор Павел (закрытый тимбилдинг 50 чел, премиум без сноба)

**Роль:** Гендиректор №15 (`32_UX_PERSONAS`). Закрытый корпоратив/тимбилдинг 50 чел, хочет премиум-опыт, но БЕЗ «элитного» сноба.
**Сценарий:** «Организую тимбилдинг 50 человек. Хочу премиальный уровень (шеф, крутая площадка, вау-эффект), но НЕ хочу чувствовать себя "плебеем", которого "снисходительно обслуживают". И чтобы быстро — без долгих звонков менеджеру.»
**Тон:** критик-персонаж, занижение. Балл ожидаемо ≤8.
**Источники:** `04_BLOCKS.md`, `02_IA.md`, `07_CALCULATOR_SPEC.md`, `08_CONSTRUCTOR_SPEC.md`, `32_UX_PERSONAS.md`, эталон `33_UXSIM_ARTYOM.md` (премиум) + `33_UXSIM_KAMILA.md` (премиум-без-сноба).

**ФОКУС ПРОВЕРКИ (гипотезы разрывов):** (а) есть ли премиум-путь без сноба (как у Артёма, но для 50, не 500), (б) не читает ли сайт «только для олигархов», (в) быстрый премиум-пресет, (г) scale-proof на 50 (не только 500), (д) value-prop «bespoke premium» без сноба.

---

## ПРОИГРЫШ СЦЕНАРИЯ (от лица Павла)

1. **Заход — ищу «премиум, но без пафоса».** Hero (`04`): «Кейтеринг, который чувствуешь заранее… под ключ для любого бюджета… без переплаты за вывеску». TrustBar — Эрмитаж/Armani/Газпром/Forbes (`04`). → Как у Артёма: средний тон («любой бюджет») + элит-логотипы = двусмысленно. Для Павла (премиум-клиент, НЕ олигарх) сайт НЕ сигналит «это для тебя» чётко, НО и НЕ снобит — ок, но нейтрально.
2. **Премиум-путь для 50.** `07` калькулятор: `PackageSelector` 4 тарифа — Эконом/Стандарт/Расширенный/**Премиум**. quick `[20][50][100]…` — 50 есть ✅. Выбираю Банкет→Премиум→50 = цена live. Но **express-пресет «премиум-тимбилдинг 50» нет** (есть svadba-80, coffee-15, corporate-120/200, волна 7 добавила банкет-500). Для Павла — 6 шагов конструктора или ручной калькулятор.
3. **Scale-proof на 50.** Артём получил scale-proof только при 300+ (`07`, волна 7/A4). На 50 никакого «мы специализируемся на премиум-тимбилдингах» нет. Павел не видит доказательства компетенции именно на его масштабе.
4. **«Без сноба» vs реальность.** Цены 50×Премиум — высокие, но без объяснения «за что платим» (шеф, площадка, вау). `WhyUs` (волна 10/B-G1) теперь несёт «чем отличаемся» — ок, НО премиум-value-prop («бесплатный шеф, персональный менеджер, white-glove») добавлен волной 7/A8 для Артёма, НЕ выделен как путь для 50-гостевого премиум-тимбилдинга.
5. **Быстрота.** Нет кнопки «Премиум-тимбилдинг под ключ →» (deep-link `?guests=50&format=banquet&tier=premium`). Павел идёт через калькулятор или звонит.

## BUG-LIST (что исправить для ≥9)

- **B-PAVEL-1 (критично, премиум-пресет 50).** Нет express-пресета «Премиум-тимбилдинг 50». *Требование:* глубокая ссылка/CTA на Hero/`/plan` `?guests=50&format=banquet&tier=premium` (как банкет-500 волны 7) — пропуск шага 1 конструктора.
- **B-PAVEL-2 (критично, scale-proof на 50–100).** Scale-proof только на 300+ (Артём). *Требование:* `ResultDisplay` при 50–150 несёт caption «Премиум-тимбилдинги на 50–150 — наша специализация / штат 80+ официантов / кейс» (расширить волну 7/A4 на средний масштаб).
- **B-PAVEL-3 (критично, value-prop без сноба).** Премиум-value-prop (шеф/менеджер/white-glove) добавлен для 500, НЕ выделен для 50-гостевого сегмента. *Требование:* в `WhyUs`/CTASection — блок «Премиум для 50 гостей: персональный шеф, выделенный менеджер, без сноба» (как волна 7/A8, но масштаб 50).
- **B-PAVEL-4 (критично, цена «за что платим»).** Высокая цена Премиум-50 без расшифровки. *Требование:* в итоге калькулятора — расшифровка премиум-составляющей (шеф, площадка, сервис), чтобы Павел видел ценность, а не только цифру.
- **B-PAVEL-5.** Нет «тимбилдинг» как типа события в `EventTypeSelector`. *Требование:* добавить карточку «Тимбилдинг / Корпоратив закрытый» (сейчас только Корпоратив/Свадьба/Выпускной/Детский/Частное/Chef-at-home) с подписью «премиум-опыт без сноба».
- **B-PAVEL-6.** `02_IA.md` не знает премиум-пресетов для среднего масштаба. *Требование:* дописать deep-link-пресеты 50/100 премиум.
- **B-PAVEL-7.** Нет быстрого контакта «мой премиум-менеджер». *Требование:* для премиум-сессии — CTA «Премиум-менеджер свяжется лично» (связь с Б2B/Агентством, B-PROFI-7).
- **B-PAVEL-8 (связь с Артёмом).** 500-люкс и 50-премиум — разные масштабы, НО общий value-prop. *Требование:* не дублировать, а расширить scale-proof на весь премиум-диапазон 20–500.
- **B-PAVEL-9 (связь с Гошей B-G).** «Без сноба» должно быть подкреплено честным сравнением (волна 10/B-G2) — Павел сравнивает с конкурентами, хочет видеть премиум-честность.

## ВЕРДИКТ (коротко)
Сайт НЕ снобит (плюс для Павла), премиум-тариф и калькулятор работают, НО нет express-пресета для 50-гостевого тимбилдинга, scale-proof только на 300+, а премиум-value-prop размазан. Павел не получает «премиум-объятия» на своём масштабе. Балл **6/10**.
