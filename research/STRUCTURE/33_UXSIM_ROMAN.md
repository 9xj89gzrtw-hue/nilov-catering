## ПРОГОН ПО СЦЕНАРИЮ (от лица Романа)

**1. Заход на главную.** Вижу Hero, TrustBar (Эрмитаж/Мариинский/Armani), потом `TestimonialsCarousel` с 3 кейсами (Анна и Павел, Северсталь-Логистика, Ирина). Читаю — цитаты бойкие, «будто мы ничего не делали сами». Глаз зацепился: в спеке прямо написано, что это **ДЕМО-плейсхолдеры, `pending-verification`, НЕ публиковать как реальные** (04 БЛОК 9, строки 536-539). То есть на живом сайте это либо скрыто, либо (по критику `30`) — «3 сочинённых цитаты». Подвох №1: первое, что видишь про отзывы, — пустой/фейковый бейдж. Не верю.

**2. Перехожу в Отзывы (`/reviews`).** В навигации desktop есть «Отзывы(/reviews)», в футере — «Компания → /reviews». Структура достижима (×). На странице: фильтр-чипы `[Все][Свадьба][Корпоратив][Частное][Chef-at-home]` → `?type=...` (SSR, удобно), grid карточек `ReviewCard`. Смотрю карточку: фото → имя → тип события → дата → меню → 1 цитата. **Нигде не написано, откуда отзыв** — самописный на сайте или с Яндекс.Карт/2ГИС/Google? Поле `source` в модели `Review` есть (04 строка 711), но `ReviewCard` его НЕ рендерит (строки 717-718). Для скептика это дыра: неотличимый самописный отзыв = «сфабриковано».

**3. Видео-отзывы.** Ищу отдельную секцию «видео-отзывы». Её нет как роута. Видео-слова гостей спрятаны в `TestimonialsCarousel variant:'video-wall'` (БЛОК 9, строки 527-534) и в `EventsRecap` (`/events/recap`). Нет чипа фильтра «С видео», нет `/reviews/video`. Нахожу только если догадаюсь скроллить карусель и там окажется video-wall. Для скептика «где видео-отзывы?» = неочевидно. Источник видео — **Rutube** (Волна 5А). Для меня Rutube авторитетнее не выглядит, чем независимые Яндекс/2ГИС, и перекрёстной ссылки «это то же лицо, что оставило отзыв на Яндексе» нет.

**4. Проверка честности.** Единственный реальный механизм верификации — `RatingBadge` (БЛОК 24): звезда + число + ссылка на Google/Yandex/2ГИС. **НО** спека говорит: если профили не заданы — плитка площадки НЕ рендерится (строка 950), а критик `30` подтверждает: сейчас `RatingBadge` **пуст**. То есть кнопки «проверить на сторонней площадке» нет вообще. Скептик остаётся заперт внутри сайта с его же самописными цитатами. Подвох №2 (самый сильный): нет пути к независимой проверке.

**Что НРАВИТСЯ (честные защиты в спеке):**
- RT-8 — дисклеймер «Отзывы с реальных мероприятий, подтверждённые фото/видео» (строка 960).
- RT-4 — `/reviews` не анонсируется как proof, пока <20 одобренных + <3 видео + <10 внеш. (строка 956).
- RT-6 — публикуют 1-2 «с замечанием» + ответ компании (честный микс, строка 958).
- RT-10 — `AggregateRating` в JSON-LD только при наличии одобренных отзывов, без хардкод «5.0» (строки 887, 951).
- RT-9 — запрет публиковать `demo`-кейсы (строки 961-963).

**ИТОГ для Романа:** архитектура *задумана* честно, но все защиты висят на дисциплине наполнения CMS, а на живом сайте (`30`) отзывам «не верят 2.5/10» — фейковые цитаты + пустой бейдж. Видимых доказательств происхождения на карточке нет. Проверить честность самостоятельно я НЕ могу.

## BUG-LIST (что исправить для ≥9)

- **[B-R1] Нет provenance на карточке отзыва.** Поле `source` ('site'|'yandex'|'2gis'|'google') есть в модели `Review` (04, строка 711), но `ReviewCard` (БЛОК 22, строки 717-718) его НЕ рендерит. Скептик не видит, самописный отзыв или с независимой площадки. → Показывать бейдж-источник + кликабельную ссылку на внешний профиль прямо на карточке (иконка Яндекс/2ГИС/Google + «Проверить отзыв»).
- **[B-R2] «Видео-отзывы» без выделенного назначения.** Спрятаны в `TestimonialsCarousel variant:'video-wall'` и `EventsRecap`; нет роута `/reviews/video` и чипа фильтра «С видео». → Добавить чип-фильтр «С видео» на `/reviews` (или под-секцию), чтобы скептик нашёл их за 1 тап.
- **[B-R3] `RatingBadge` (БЛОК 24) пуст = нет внешней верификации.** Главный механизм проверки честности для скептика недоступен (критик `30`: бейдж пуст). → Обязать заполнение ≥1 внеш. профиля (yandex/twoGis/google) ДО публикации `/reviews` как proof; увязать с метрикой готовности RT-4.
- **[B-R4] `/reviews` недостижим из `MobileBottomNav`.** 5 пунктов (События/Меню/Галерея/Почему мы/Спланировать) без «Отзывы» — только через бургер. → Либо вынести «Отзывы» в видимый якорь на главной/в bottom-nav, либо добавить чип в бургер-меню как приоритет.
- **[B-R5] Риск публикации `demo`-кейсов не перекрыт технически.** В спеке — только текстовый запрет «НЕ публиковать» (RT-9), но выдача фильтруется только по `approved`, флаг `demo` не отсекается при рендере. → Фильтровать выдачу `approved === true && !demo` на уровне компонента/запроса.
- **[B-R6] Поле `reply` (ответ компании, RT-7) не выделено на карточке.** В модели есть (строка 712), в составе `ReviewCard` (717-718) не упомянуто. Живой ответ компании = сильный сигнал реальности. → Рендерить блок «Ответ NiloV» под цитатой.
- **[B-R7] Видео-отзывы на Rutube без cross-link на внешний отзыв.** Для скептика Rutube менее авторитетен, чем Яндекс/2ГИС; нет связи «это то же лицо, что оставило отзыв снаружи». → Добавить перекрёстную ссылку videoTestimonial → внешний отзыв того же клиента (поле `source` + `clientName`).

**Позитив (не ломать):** RT-4 (объём ≥20/≥3 видео/≥10 внеш.), RT-6 (честный микс +/-), RT-8 (дисклеймер «реальные отзывы»), RT-10 (без фейк-AggregateRating) — держать как конкурентное преимущество честности.
