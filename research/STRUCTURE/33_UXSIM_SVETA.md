# 33 — UX-симуляция: Мама младенца Света (детский праздник, коляска/аллергии)

**Роль:** критик-персонаж №9 (`32_UX_PERSONAS`). Настроен на занижение.
**Сценарий:** детский праздник, ребёнок-аллергик (на глютен), с коляской → найти аллергены/диету по каждому блюду, понятно ли всё для мамы.
**Что прочитано:** `04_BLOCKS.md`, `02_IA.md`, `08_CONSTRUCTOR_SPEC.md`, `32_UX_PERSONAS.md`.

> **Важная оговорка (версия спеки).** Симуляции Ольги и Виктора (`33_UXSIM_OLGA/VIKTOR`) играли СТАРУЮ IA, где «Детский праздник» был спрятан под «Частное». В ТЕКУЩИХ спеках (маркеры «Волна 1» / B-OLGA-1…7) детский путь УЖЕ первый-class: `/events/detskoe`, `/menu/detskoe`, `childFriendly`-бейдж, аниматор-addon, детский прайс. Поэтому по структуре Света устроена лучше Ольги. НО её ГЛАВНАЯ потребность — аллергены — упирается в незакрытый блокирующий баг: **данные 14 аллергенов пусты для всех 78 блюд** (R1, `25_MENUCRITIC_FINAL`). Ниже — проигрыш именно по актуальным спекам.

---

## Проигрыш сценария (шаг за шагом)

**1. Заход на `/` (одной рукой, вторая на коляске = mobile, одноруко)**
Hero: «Кейтеринг, который чувствуешь заранее» + микро-строка «Маркировка по 14 аллергенам» (`04`#1, стр. 297). `EventTypeSelector` — ровно **5** карточек: Свадьба / **Детский праздник** / Корпоратив / Частное / Chef-at-home (B-OLGA-1 закрыт). У детского — цена «от 1 950 ₽/гость» и ссылка `/events/detskoe`.
→ Плюс: детский праздник НЕ спрятан, карточка на виду. Минус: на коляске Света в один тап лишь попадает на посадочную — про аллергены там НИ СЛОВА (см. шаг 2).

**2. Детский праздник → `/events/detskoe`**
`EventHero` (`04`#14, стр. 620-621): фото, теги «угощения для детей / аниматор / детское меню», CTA «Собрать детский праздник» → `/plan/constructor?event=detskoe`.
→ Никакого блока про **аллергии / безглютеновое / диету**. Ни предупреждения, ни якоря «меню с пометкой аллергенов». Света не видит, что её вопрос вообще учтён. Скрытый раздел внутри детского пути.

**3. Меню → `/menu/detskoe` (детское меню)**
Роут есть (`02_IA` стр. 29, B-OLGA-2). Показывает детские блюда (`DishCard` с `childFriendly`+бейдж «Детям 🧒»). На карточке — `dietBadges` (`gluten-free` и пр.) + чипы 14 аллергенов (`04`#17, стр. 1250-1281).
→ **НО:** (а) на `/menu/detskoe` НЕТ явного фильтра «без глютена» — он только в общем `/menu/catalog` (чип `gluten-free`, стр. 1200). Чтобы отсеять глютен, Света должна уйти с детского меню в каталог и СКОМБИНИРОВЬ два чипа (`детское` + `gluten-free`) — неочевидно и не в один тап (на коляске — мучительно). (б) **Данные аллергенов пусты** (R1): чипы `allergens[]` и `crossContact` не заполнены ни для одного из 78 блюд → «маркировка по 14 аллергенам» из Hero — пустая декларация. Света видит бейджи-пустышки.

**4. Аллергены / диета (где правда?)**
В `MenuCatalog`: фильтр по диете (vegan/vegetarian/gluten-free/halal/детское — счётные подборки, `04`#16, стр. 1190-1202) + на `DishCard` — 14 аллергенов и `crossContact`-чип. В конструкторе (шаг 3, B-MUX-4) — per-dish «исключить аллерген X» (`08` стр. 87-90).
→ Теоретически мощно. ПРАКТИЧЕСКИ для мамы-аллергика:
- **нет plain-language вердикта** «подходит ребёнку-аллергику / НЕ подходит» — только технические названия 14 аллергенов ТР ТС (глютен, ракообразные, рыба…). Для встревоженной мамы опасность не проговорена.
- **crossContact** (перекрёстная контаминация на кухне) критичен для младенца, но это лишь тонкий чип «⚠ перекрёстная контаминация», не вынесен красным предупреждением наверх детского меню, и опять же — данных нет.
- на **mobile** аллергены раскрываются на hover/focus (`04`#17, стр. 1279); на тач-устройстве hover нет → надо тыкать/скроллить каждую карточку, одной рукой с коляской — неудобно.

**5. Детский пресет в конструкторе**
`format='detskoe'` авто-добавляет аниматора + фильтрует меню на `childFriendly` (`07` стр. 152-153). Но фильтр `childFriendly` ≠ «без глютена»: детское блюдо может содержать глютен. Авто-фильтр по аллергии НЕ срабатывает — Свете снова вручную ковырять исключения аллергена в шаге 3.

## Оценка по 4 осям (0–10, занижение)

| Ось | Балл | Почему |
|---|---|---|
| 1. Понятность структуры | **6** | Детский путь теперь первый-class (`/events/detskoe`, `/menu/detskoe`, карточка в `EventTypeSelector`) — НЕ сирота. Но аллергия/диета НЕ связана с детским флоу: ни на `/events/detskoe`, ни на `/menu/detskoe` нет якоря «аллергены/безглютеновое»; фильтр глютен только в каталоге. |
| 2. Удобство действий | **4** | Найти детское меню — легко. Найти «без глютена» именно в детском — надо уйти в `/menu/catalog` и скомбинировать 2 чипа; данные пусты → результат иллюзорен. Per-dish исключение есть, но только в шаге 3 конструктора. Для мамы с коляской (одна рука) — слишком много тапов. |
| 3. Дискомфорт/перегрузка | **4** | 14 технических аллергенов без plain-language вердикта «подходит/не подходит»; crossContact спрятан в тонкий чип; Hero обещает «маркировку по 14 аллергенам», а по факту данных нет → тревога и недоверие. Перегруз терминами при дефиците реальной информации. |
| 4. Устройства | **5** | Mobile-костяк неплох (5 пунктов bottom-nav, sticky CTA, тач ≥44px, reduced-motion). НО аллергены раскрываются на hover (не работает на тач → тап/скролл каждой карточки); однорукий фильтр из 2+ чипов неудобен; на коляске путь тяжёлый. |

**Средний балл: 4.75 / 10** (ожидание ≤8 выполнено).

## Bug-list (что исправить для ≥9)

1. **B-SVETA-1 (КРИТИЧНЫЙ, данные):** 14 аллергенов и `crossContact` пусты для всех 78 блюд (R1, `25_MENUCRITIC_FINAL`). Пока данные не заполнены из карточек заказчика — «маркировка по 14 аллергенам» (Hero, TrustProof) = пустая декларация, опасная для мамы-аллергика. Закрыть ДО прода (bulk-импорт, `29_CMS_CRITIC` B-CMS-3/R-E).
2. **B-SVETA-2 (фильтр на детском меню):** на `/menu/detskoe` НЕТ явного чипа «без глютена» / «без аллергенов». Добавить diet-фильтр прямо на детскую страницу (не заставлять уходить в `/menu/catalog` и комбинировать `детское`+`gluten-free`). Быстрый пресет «Детское меню БЕЗ глютена» в 1 тап.
3. **B-SVETA-3 (crossContact / безопасность):** для младенца перекрёстная контаминация критична. Вынесити наверх детского меню/карточки красное предупреждение «⚠ на кухне возможен контакт с глютеном» когда `crossContact:true`, а не прятать в тонкий чип.
4. **B-SVETA-4 (plain-language вердикт):** вместо сухих 14 аллергенов ТР ТС дать понятный вердикт по блюду: «✅ подходит при аллергии на глютен» / «❌ содержит глютен — не для ребёнка-аллергика». Язык мамы, не ГОСТа.
5. **B-SVETA-5 (mobile/hover):** аллергены раскрываются на hover (`04`#17) — на тач не работает. Сделать чипы 14 аллергенов + diet-бейджи **всегда видимыми на карточке** (не только hover/focus), чтобы не тықать каждую. Крупные тач-цели фильтра для однорукого режима с коляской.
6. **B-SVETA-6 (пресет конструктора):** детский пресет фильтрует на `childFriendly`, но НЕ на аллергию. Добавить в пресет опц. шаг/чекбокс «у ребёнка аллергия: ___» → авто-фильтр меню по `gluten-free`/`allergens` + авто per-dish exclude, чтобы Свете не ковырять шаг 3 вручную.
7. **B-SVETA-7 (якорь на `/events/detskoe`):** на посадочной детского праздника добавить блок «Аллергии и диета учтены» с коротким объяснением (14 аллергенов, безглютеновые опции, crossContact) и ссылкой прямо на отфильтрованное детское меню — иначе Света не верит, что её случай вообще видят.

---

*Симуляция завершена. Резюме: структура детского праздника в актуальных спеках исправлена (B-OLGA-1…7) и для Светы лучше, чем для Ольги, — НО её главная потребность (аллергены/диета) бьётся о блокирующий разрыв: данные 14 аллергенов пусты (R1), фильтр «без глютена» не вынесен на детское меню, crossContact и plain-language вердикт отсутствуют, hover-раскрытие аллергенов не работает на тач. Балл 4.75/10.*
