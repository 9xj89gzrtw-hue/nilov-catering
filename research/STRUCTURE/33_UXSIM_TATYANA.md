# 33 — UX-СИМУЛЯЦИЯ: HR Татьяна (корпоратив 200 чел)

**Роль:** HR-менеджер Татьяна, корпоратив на 200 человек. Критик-персонаж, занижение.
**Сценарий:** «Организую корпоратив на 200 человек. Хочу получить расчёт онлайн, загрузить бриф, понять банкет vs фуршет, не звонить лишний раз».
**Проигрыш:** заход → корпоратив → сравнение банкет/фуршет → расчёт 200 чел → загрузка брифа.
**Источники:** `04_BLOCKS.md`, `02_IA.md`, `08_CONSTRUCTOR_SPEC.md`, `32_UX_PERSONAS.md`.

---

## ПРОХОД ПО ШАГАМ (голос Татьяны)

### 1. Заход (главная `/`)
Герой: одна кнопка «Спланировать событие» → `/plan`. Понятно. Ниже — 4 карточки событий, в т.ч. «Корпоратив» → `/events/korporativ`. Навигация 8 пунктов + тел/WA/ТГ. Перегрузка средняя, но логично.
**Что почувствовала:** ок, куда жать — понятно.

### 2. Корпоратив (`/events/korporativ`)
Хаб события → «рекомендованные форматы из /menu». По идее ведёт на `/menu/furshet` и `/menu/banquet`. Контекст корпоратива есть, но **нет кнопки «запросить расчёт на 200»** — только общие рекомендации.
**Разрыв:** страница даёт «почитай про форматы», но не даёт действия для моего размера.

### 3. Сравнение банкет / фуршет
**Здесь провал.** Нет инструмента «банкет vs фуршет рядом». `FormatShowcase` на главной даёт только «от X ₽/гость» (и цены битые, см. баги). Чтобы сравнить — открывать `/menu/banquet` и `/menu/furshet` **в два захода**, читать про посадку/официантов/станции по отдельности. Для решения на 200 чел это критично, а инструмента нет. Можно прогнать калькулятор дважды, но это «вслепую».

### 4. Расчёт 200 чел (`/plan/calculator` или `/plan/constructor`)
Путь: «Спланировать» → `/plan` (хаб) → `/plan/calculator` — **два лишних хопа**, прямой CTA «Рассчитать 200» нет.
- Калькулятор: 200 в диапазоне 10–500, считает. **Но quick-кнопки гостей `[20][50][100][150][300]` — 200 НЕТ.** Пришлось тыкать слайдер/вбивать вручную. Для корпоратива 200 — базовое число, почему его нет?
- Конструктор: 6 шагов, включая «собрать свою линию по станциям» (B2C-штука). **HR не хочет собирать меню порциями** — ей нужна смета + бриф. Перегрузка.
- Лайв-итог на каждом шаге — хорошо.

### 5. Загрузка брифа
**ПОЛНЫЙ ТУПИК. Критично.**
У меня есть ТЗ корпоратива (площадка, список аллергий, тайминг, бюджет) в файле. Где его загрузить?
- `ContactForm`: имя*, телефон*, дата*, **коммент** — поля ввода, **поля для файла НЕТ**.
- Конструктор шаг 6 (Контакт): те же поля + коммент. Файл не принимается.
- Ветка «Свяжитесь с менеджером» → `/api/contact` «передаёт контекст из State» (формат/гости/дата/add-ons) — **но файл брифа не прикрепляет**.
**Итог:** загрузить бриф невозможно нигде. Придётся либо перепечатывать ТЗ в комментарий (потеря данных, ошибки), либо звонить. Мой сценарий «не звонить + загрузить бриф» сломан по определению.

## ОЦЕНКА ПО 4 ОСЯМ (0–10, занижение, ≤8)

| Ось | Балл | Почему |
|---|---|---|
| 1. Понятность структуры | **7** | Логика есть (герой→тип→форматы→план), но «бриф» и «сравнение» — сироты без дома; `/api/contact` берёт State, но не файл. |
| 2. Удобство действий | **5** | Расчёт 200 рабочий, но: 2 хопа до калькулятора, 200 нет в quick-кнопках, загрузить бриф нельзя (нужен звонок). |
| 3. Дискомфорт/перегрузка | **5** | 6-шаговый конструктор для HR — оверкилл; нет сравнения форматов; конфликтующие «от X ₽» между главной и меню. |
| 4. Устройства (mobile/tablet/desktop) | **7** | Sticky-CTA и bottom-nav хороши, но ведут только в `/plan`, не в калькулятор; на мобиле 6 шагов конструктора тяжелы; загрузки брифа нет нигде. |

**Средний балл: 6.0 / 10** (ожидаемо ≤8). Главный провал — ось 2 (действия) из-за отсутствия загрузки брифа.

## BUG-LIST (что исправить для ≥9)

- **B-TAT-1 (БЛОКИРУЮЩИЙ).** Добавить **загрузку файла брифа** (PDF/XLS/DOC, ≤10 МБ) в `ContactForm` и в шаг 6 конструктора; поле «Загрузить бриф» → `/api/contact` с вложением. Без этого сценарий Татьяны невыполним.
- **B-TAT-2.** Сделать **инструмент сравнения «Банкет vs Фуршет»** (в две колонки: посадка/официанты/цена за гостя/вау-опции), достижимый из `/events/korporativ` и `/menu`.
- **B-TAT-3.** Добавить **[200]** (и [500]) в быстрые кнопки гостей калькулятора/конструктора — 200 для корпоратива канонично.
- **B-TAT-4.** Устранить **рассинхрон цен** (BUG-F2): единый источник «от X ₽/гость»; не показывать конфликтующие полы между `FormatShowcase`, `/menu/*` и калькулятором (сейчас 2 450 vs 390).
- **B-TAT-5.** Прямая CTA **«Рассчитать 200 гостей»** с `/events/korporativ` → `/plan/calculator?guests=200&format=banquet` (предзаполнение). Убрать лишний хоп через хаб.
- **B-TAT-6.** Лёгкий **B2B-путь «Быстрый расчёт + бриф»**: калькулятор + загрузка брифа без 6-шагового конструктора меню (HR не собирает линию блюд).
- **B-TAT-7.** Mobile sticky-CTA: опц. «Рассчитать» рядом с «Спланировать» (прямая ссылка на калькулятор, не через `/plan`).
- **B-TAT-8.** Ветка «Свяжитесь с менеджером» должна **передавать и файл брифа**, а не только State.
