## BUG-LIST по СВОЕЙ ЗОНЕ (цвет / типографика / геометрия / консистентность токенов)

> Все контрасты измерены по WCAG 2.1 (sRGB, отношение яркостей). Цитаты — из `01_VISUAL_DNA.md` / `06_TOKENS.md`.

### B1 — Заявленный контраст золота ЛОЖЕН (блокирующий a11y)
- **Где:** `01` §2, строка 39: *«Акцент 1 | Champagne gold | `#B08D57` … 4.6:1 на ivory ✓ AA»*; используется для *«CTA, цены, бейджи, разделители»* как текст.
- **Факт:** измеренный контраст `#B08D57` на `#FAF7F2` = **2.89:1**. Провал AA для всего обычного текста (нужно ≥4.5). Золото — самый частый цвет текста на сайте, и он нечитаем для слабого зрения.
- **Бенчмарк:** Proof of the Pudding / awwwards food-лидеры держат золото как ЗАЛИВКУ на тёмном или как тонкую линию, никогда — как цвет body/CTA-текста на креме.
- **Фикс:** для текста на светлом затемнить золото до ~`#8A6D3B` (≈4.5:1) либо оставить золото только на тёмных заливках.

### B2 — Пале-голд для ЦЕН катастрофически низкий (блокирующий)
- **Где:** `06` §1, строки 44–45: *«--color-gold-soft: #C9A961; /* цифры/цена (ярче на светлом) */»*; маппинг §5: *«PriceTag | font-mono, text-gold-soft»*.
- **Факт:** `#C9A961` на ivory = **2.11:1**, на white = **2.25:1**. Цена — самый конверсионно-критичный текст кейтеринга — практически невидима для слабого зрения. Комментарий «ярче на светлом» фактически ОШИБОЧЕН: ярче = ниже контраст.
- **Бенчмарк:** топ-catering-сайты рендерят цены в высококонтрастном ink, золото — декоративная подчёркивающая линия, не сам текст.
- **Фикс:** цены в espresso/ink (`#1C1815`), золото — только как декоративный разделитель/underline под ценой.

### B3 — White-on-gold кнопка проваливает AA обычного текста
- **Где:** `06` §5, строка 161: *«Button (primary) | bg-primary (#B08D57), text-primary-foreground (#FFFFFF), rounded-full…»*.
- **Факт:** white на `#B08D57` = **3.09:1**. Проходит AA только для крупного/жирного (≥18.66px bold или ≥24px). Если primary-CTA 14–16px белым — FAIL.
- **Фикс:** затемнить gold до ~`#8A6D3B` (white-on-~4.3:1) ИЛИ использовать тёмный ink-текст на золоте.

### B4 — Ромбовидная рамка — хак через rotate-45 (реализация сломается)
- **Где:** `06` §3, строки 119: *«--radius-diamond: 8px; /* rotate-45 → ромб */»*; `01` §5: *«Круглые и ромбовидные фреймы»*.
- **Факт:** `rotate-45` на прямоугольном контейнере фото поворачивает ВЕСЬ бокс (вместе с контентом/текстом) → перекос изображения и невозможность вложить подпись без обратного поворота. «Ромб» через radius-токен вообще не получается — radius не делает ромб.
- **Бенчмарк:** Queen of Hearts / Proof of the Pudding используют НАСТОЯЩИЕ маски (`border-radius:50%` + `object-fit:cover` для круга; `clip-path: polygon(...)` для ромба).
- **Фикс:** ромб через `clip-path: polygon(50% 0, 100% 50%, 50% 100%, 0 50%)` + `object-fit:cover`; либо убрать ромб, оставить круг.

### B5 — Sage «в сезоне»: контраст не задан и почти наверняка FAIL
- **Где:** `01` §2 Sage `#7E8B6B` (*«успех/сезон … eco-метки»*); `06` `--color-success:#4F7A52` / `--color-sage:#7E8B6B`.
- **Факт:** ни в `01`, ни в `06` нет проверки контраста sage на ivory/white. Оценка: sage `#7E8B6B` на ivory ≈ **2.8–3.2:1** → провал AA для текста. Eco-метки («в сезоне») окажутся блёклыми.
- **Бенчмарк:** экологичные/сезонные лейблы у лидеров держат ≥4.5:1 (тёмный текст по светлому sage-фону, не sage-текст по белому).
- **Фикс:** sage — только как фон бейджа с тёмным текстом (`#1C1815`/`#4A423B`); не как цвет текста.

### B6 — Премиум-утечка в ДНК не закрыта (рассогласование токены vs DNA)
- **Где:** `01` §1 *«деликатное золото как знак премиальности»*; §7 *«Looks expensive чек-лист»*; §2 burgundy *«премиум-бейджи»*. TrustSegCritic (29) выписал T1/T2/T10 — переименовать в «тепло/праздник/опрятность», «Looks trustworthy», «хит/избранное».
- **Факт:** токен-слой `06` даже поднял muted до AAA (`#4A423B`) — то есть исполнительный слой дисциплинирован и middle-market-чист, а нарративный слой ДНК ВСЁ ЕЩЁ продаёт премиум. Две правды.
- **Бенчмарк:** hooman.com (2025) «Expensive isn't loud. It's calm»; MobilCard (2026) глянец может подорвать посыл. Середина читает «премиум» = «не для меня» (29: соответствие 4/10).
- **Фикс:** применить T1/T2/T10 к `01` ДО сборки — иначе компоненты получат премиум-лексику из ДНК.

### B7 — Золотой overline делает иерархический «бровь» самым слабым элементом
- **Где:** `06` §5, строка 167: *«SectionHeading | overline text-2xs uppercase text-gold»*.
- **Факт:** 12px uppercase золото `#B08D57` на ivory = **2.89:1**. Eyebrow (надзаголовок) по хорошей практике должен быть тихим, но ЧИТАЕМЫМ; здесь он — самый низкоконтрастный элемент секции, что ломает иерархию (ось 1).
- **Фикс:** overline в earth `#7C6A55` (5.1:1) или ink-soft `#4A423B` (9.2:1); золото — только для разделителя под overline.

### B8 — «Живые фото» внутри круг/ромб-масок не продуманы (риск убить УТП)
- **Где:** `01` §5 / `06` §3: круглые/ромбовидные фреймы с 1px gold + Elevation-1; `01` §1 требует «каждое фото живое и что-то делает» (Ken Burns/loop-видео).
- **Факт:** Ken Burns zoom (1.0→1.08) и loop-видео ВНУТРИ круглой/ромбовидной маски требуют `object-fit:cover` + чёткого `object-position` и корректного `overflow:hidden` — иначе «живой» кадр обрезается по важному (тарелка/лицо). Ни маска, ни object-position не заданы как токены.
- **Бенчмарк:** PoTP держит full-bleed ВИДЕО без рамок; SOTD-лидеры маскируют круглое видео аккуратно (object-position задан явно).
- **Фикс:** завести токены `--frame-object-pos` и протестировать Ken Burns/видео внутри каждой маски на реальных кадрах.

### B9 — Типографическая пара компетентна, но «безлика» (не уровень «лучший в мире»)
- **Где:** `01` §3 / `06` §2: Cormorant Display + Inter + JetBrains Mono.
- **Факт:** Cormorant — самый часто используемый сериф на awwwards (буквально десятки food-сайтов). Пара «serif-заголовок + Inter + моно-цены» корректна и в тренде (Nice Branding 2026: «Elegant Serif returns», «2–3 families»), но не отличима. JetBrains Mono для цен — модный, но на кейтеринге чуть натянут (техно-лексика).
- **Бенчмарк:** awwwards food-лидеры 2025–26 идут на кастомный/display-сериф (высококонтрастный Didone или характерный humanist) для H1 + humanist-sans для UI, иногда 3-й выразительный шрифт.
- **Фикс:** для H1 взять менее избитый display-сериф (характерный, не Playfair); Inter оставить для UI; моно — только для цифр калькулятора, не для всех цен.

### B10 — Контраст-аннотации в spec ненадёжны (gov-баг слоя спецификации)
- **Где:** `01` §2: gold «4.6:1», burgundy «6.3:1». Измерено: gold **2.89**, burgundy **9.03**.
- **Факт:** собственные «✓ AA»-печати spec нельзя доверять — gold занижен в 1.6×, burgundy завышен. Кто-то строит против документа с ложными гарантиями доступности.
- **Фикс:** пересчитать контраст для ВСЕХ 10 ролей палитры, в spec оставить только верифицированные значения с датой замера.
