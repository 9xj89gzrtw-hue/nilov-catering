## Правила (anti-self-bias, из skill `self-critic-system`)

1. **Автор (основной агент) НЕ ставит баллы** — только чинит bug-list.
2. Критики — независимые субагенты (qwen3-coder), настроены ЗАНИЖАТЬ.
3. Каждый критик ОБЯЗАН использовать web (сравнение с реальными мировыми лидерами) —
   иначе «отлично» = «совпадает с моим воображением».
4. Любой балл ≥8 требует PROOF (цитата спеки + ссылка на мировой пример) или падает до 7.
5. Веса/пороги заморожены после старта оценки.
6. ≥3 критических дизайн-бага у одного критика → авто-REWORK.

## Вердикты
- **PASS:** средневзвешенный ≥8.0 И нет оси <5.
- **REWORK:** 6.0–7.9 ИЛИ любая ось <5.
- **FAIL:** <6.0.

## Формула Judge
Средний дизайн-балл = Σ(ось_i × вес_i), усреднённый по 3 критикам (A-foundation,
B-blocks, C-motion-media). Затем цикл: автор чинит спеку по bug-list → повторная
критика → пока средний не **> 8.0**.
