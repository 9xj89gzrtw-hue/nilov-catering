## 4. Bug-list (≥10, формат: `file:line` + альтернатива)

> Контраст пересчитан по формуле WCAG (sRGB→linear→L, отношение (L1+0.05)/(L2+0.05)).

**BUG-1 · Золото как ТЕКСТ/акцент на светлом — 2.89:1 (FAIL 1.4.3 и 1.4.11).**
`04_BLOCKS.md:33,47,254` и `04_BLOCKS.md:1463` (`SectionHeading overline text-gold-text`).
`#B08D57` на `#FAF7F2` = **2.89:1**. Overline «КЕЙТЕРИНГ…», разделители, акцентные линии, ссылки используют gold как текст/обязательный UI-цвет.
*Альтернатива:* текстовые акценты — только `ink #1C1815`/`earth #7C6A55` (4.85:1, AA). Gold оставить ТОЛЬКО для не-текстовых декоративных элементов ≥24px ИЛИ затемнить акцентный gold до `#8A6D3B` (≈4.6:1 на ivory) для использования как текст.

**BUG-2 · `gold-soft #C9A961` на светлом — 2.11:1 (FAIL).**
`04_BLOCKS.md:48`. Используется как мягкий акцент/деталь.
*Альтернатива:* не использовать `gold-soft` для текста/границ; либо заменить на `#8A6D3B`+.

**BUG-3 · Focus-visible ring gold на ivory — 2.89:1 (FAIL 1.4.11, нужно 3:1).**
`04_BLOCKS.md:42,225` (`--color-ring:#B08D57`, `*:focus-visible{outline:2px solid var(--color-ring)}`).
Кольцо фокуса — это UI-компонент, требует 3:1. На ivory 2.89 — не проходит. (На белой карточке `#FFFFFF` gold = 3.09 — едва проходит, но база сайта ivory.)
*Альтернатива:* `--color-ring:#8A6D3B` (≈4.6:1) ИЛИ двойное кольцо `outline:2px solid #B08D57; outline-offset:2px` + подложка `box-shadow:0 0 0 4px #1C1815` для гарантии 3:1 на любом фоне.

**BUG-4 · A2 требует `transcript` для видео, но схема `VideoClip` поле НЕ содержит (самопротиворечие).**
Требование: `04_BLOCKS.md:253` («CMS-поле `transcript`/`description` ОБЯЗАТЕЛЬНО для `VideoClip`»).
Схема: `04_BLOCKS.md:91` (поля `videoUrl, poster, eventType, title, durationSec` — **нет transcript/description**).
*Альтернатива:* добавить в `VideoClip` обязательные `transcript: string` и `description: string`; `VideoFacade` рендерит ссылку «Текстовая версия» + `aria-describedby`.

**BUG-5 · A7 требует обязательный `alt` фото, но `Dish.photo` не имеет поля `alt` (самопротиворечие).**
Требование: `04_BLOCKS.md:258` («`Dish.photo.alt` … обязательное поле»).
Схема: `04_BLOCKS.md:88` (`photo (image, req)` — поле `alt` не описано).
*Альтернатива:* в схеме `Dish` явно `photo: { src, alt: string (req, не имя файла) }`; валидация «ни одно из 78 фото не уходит с пустым alt» (уже в A7 — перенести в схему).

**BUG-6 · `HomeVideoShowcase` autoplay-по-IO противоречит reduced-motion-гейту (вероятный autoplay под reduced-motion).**
`04_BLOCKS.md:478` («Блок 27 (`HomeVideoShowcase`) НЕ трогаем — его autoplay-по-IO») vs `04_BLOCKS.md:714` («`prefers-reduced-motion` → НЕ автозапуск»).
Примечание 478 явно говорит «не трогаем», то есть reduced-motion-обработка, добавленная в 714, в блок 27 не перенесена.
*Альтернатива:* явно прописать, что IO-autoplay блока 27 НАХОДИТСЯ под тем же гейтом `prefers-reduced-motion` (показ poster + кнопка play).

**BUG-7 · Бегущая строка (marquee TrustBar) — нет остановки с клавиатуры/для SR (FAIL 2.2.2).**
`04_BLOCKS.md:26` («marquee плавный, hover-pause»), `04_BLOCKS.md:537` (CSS transform + rAF), `04_BLOCKS.md:539` (`<ul aria-label>`).
Требование WCAG 2.2.2: движущийся контент >5с должен быть останавливаем наведением ИЛИ фокусом. Спека гарантирует только `hover-pause` — для клавиатуры/SR пользователя остановки нет.
*Альтернатива:* добавить `prefers-reduced-motion` → marquee статичен; либо кнопку паузы/остановки с фокусом; либо паузу по `:focus-within` контейнера.

**BUG-8 · SkipLink / якорная навигация не работает под Lenis (focus есть, viewport не движется).**
`04_BLOCKS.md:128` («SkipLink на контент»), `04_BLOCKS.md:158` (`SmoothScroll(Lenis)` обёртка всего сайта).
Lenis перехватывает скролл; программный переход по `#main`/якорю (SkipLink) перемещает фокус, но viewport может не прокрутиться к цели.
*Альтернатива:* на `SkipLink` и всех hash-ссылках вызывать `lenis.scrollTo(target)`; либо при нажатии SkipLink временно `lenis.stop()` + нативный `scrollIntoView`.

**BUG-9 · focus-trap ВНУТРИ mega-menu навигации — антипаттерн, риск запирания фокуса.**
`04_BLOCKS.md:120` («focus-trap внутри» mega-menu «События»/«Меню»).
Жёсткий focus-trap в навигационном меню — это не модалка; он может поймать клавиатурного пользователя, который хочет выйти из меню к основному контенту (нужен `Escape`→закрыть+вернуть фокус, а не «ловушка»).
*Альтернатива:* roving tabindex внутри меню + `Escape`/`Tab` на последнем элементе закрывает меню и возвращает фокус к триггеру; не использовать `inert`/trap на всю страницу.

**BUG-10 · Бейдж «Популярный»: earth `#7C6A55` на `gold-tint #EFE6D6` = 4.19:1 (FAIL 1.4.3 для обычного текста).**
`04_BLOCKS.md:1570` (`bg-gold-tint text-earth`).
4.19:1 проходит только для крупного текста (≥18.66px bold / 24px). Бейдж — мелкий лейбл.
*Альтернатива:* текст бейджа `ink #1C1815` на `gold-tint` (≈9:1) ИЛИ затемнить earth до `#5C4E3E` (≈5:1).

**BUG-11 · `accent-foreground #7C6A55` на `accent #EFE6D6` = 4.19:1 (FAIL для обычного текста).**
`04_BLOCKS.md:39` (`--color-accent-foreground:#7C6A55` на `--color-accent:#EFE6D6`); используется как текст акцентных блоков.
*Альтернатива:* `accent-foreground:#5C4E3E` (≈5:1) либо использовать accent-фон только для крупного текста.

**BUG-12 · `warning #B5803A` (3.21:1) и `sage #7E8B6B` (3.39:1) на ivory — FAIL 1.4.3, если используются как текст.**
`04_BLOCKS.md:55,56`; упоминаются как статусные цвета (warning-тон, sage). `04_BLOCKS.md:1722` говорит «текст ink/earth на tint ≥4.5» — но сами токены `warning`/`sage` как текст на ivory не проходят.
*Альтернатива:* статусный текст — `ink`/`earth`; `warning`/`sage` оставить только для иконок/полос ≥3:1 (1.4.11) или затемнить (`#8A5A1E`, `#54633F`).

**BUG-13 · `<html lang>` нигде не mandated (FAIL 3.1.1).**
По всему `04_BLOCKS.md` (и прочим файлам commission) нет требования проставить `lang="ru"` на `<html>`. Screen-reader не знает язык страницы.
*Альтернатива:* жёсткое требование в Фазе 0 / `layout.tsx`: `<html lang="ru">` (и `lang` на иноязычных вставках).

**BUG-14 · Формы (ContactForm/QuoteForm/Constructor) — ассоциация label/aria-invalid/aria-describedby не прописана.**
`04_BLOCKS.md:141-144` (ContactForm, только «нужные поля + 152-ФЗ чекбокс»), ссылки на `07`/`08`. Нет контракта: `<label for>`, `aria-invalid`, `aria-describedby` на ошибках, `aria-required`.
*Альтернатива:* обязать в спеце `07`/`08` явные `<label>` + `aria-invalid` + `role="alert"` на сообщениях ошибок + `aria-describedby` на hint.

**BUG-15 · CustomCursor переиспользуется — риск 2.5.7 и скрытия нативного курсора.**
`04_BLOCKS.md:32` (переиспользовать `components/effects/*` включая `CustomCursor`). Кастомный курсор может скрывать нативный, мешать пользователям с тремором (dragging) и low-vision.
*Альтернатива:* убрать CustomCursor из светлой сборки (или `pointer:fine` + никогда не прятать нативный курсор); не использовать для drag-жестов без эквивалента кнопкой.

**BUG-16 · A3 (254) ложно заявляет «Контраст (B1) закрыт / AAA» — вводит в заблуждение по headline-дефекту.**
`04_BLOCKS.md:254` («токены применены … `--color-muted-foreground:#4A423B` ≈8.9:1, AAA. ЗАКРЫТО»).
Утверждение верно ТОЛЬКО для muted-foreground. Gold (центральный бренд-акцент) как текст/UI = 2.89:1 — НЕ закрыт. Self-claim противоречит фактам BUG-1/2/3.
*Альтернатива:* в A3 явно написать «gold как текст на ivory FAIL; требуется затемнение/замена» и снять статус «закрыто» по B1 до правки токенов.

## 6. Вердикт

**Итоговый балл доступности спецификации: 5.0 / 10 (строго ≤9).**

**Почему не выше:**
1. **Фундаментальный дефект ДНК:** золотой — центральный бренд-акцент premium-светлой ДНК — на ivory даёт 2.89:1. Он используется как текст (overline, headings), акценты, разделители и **focus-ring**. Это бьёт сразу 1.4.3 (текст) и 1.4.11 (UI-компоненты). Исправление = перекрасить акцентный gold или запретить его как текст — ломает визуальную ДНК, которую критик не может «просто одобрить».
2. **Самопротиворечия в самой спеце** (BUG-4, BUG-5, BUG-16): a11y-требования (transcript, alt) заявлены, но CMS-схемы их не несут; A3 ложно клеймит «закрыто». Спеце нельзя верить на слово.
3. **Контракты не исполнены** (Фаза 0 не собрана, `41`): focus-ring/touch/reduced-motion опираются на `globals.css`, который остаётся legacy-dark. До сборки «AA» — это обещание, не факт.

**Почему не ниже (3–4):** спецификация исключительно детальна именно по a11y — блоки A1–A9 (Анна), K1–K9 (планшет), B11–B13 + D4–D8 (mobile), явный reduced-motion для Lenis/прелоадера/видео, aria-live для цен, radiogroup для тарифов, focus-trap для модалок, 44px touch-targets. Это редкий для промо-сайта уровень проработки намерений.

**Главный блокирующий пункт перед сборкой:** пересмотреть токены золотого акцента (BUG-1/2/3/10/11/12) и привести CMS-схемы в соответствие с a11y-требованиями (BUG-4/5), иначе сборка по `41` Фазе 0 запишет FAIL-контраст в код, а «axe 0 critical» (41:168) не пройдёт.

**Объективный якорь для приёмки (из `19`/`41`):** Lighthouse a11y AA + axe 0 critical. При текущих токенах axe **гарантированно** выдаст critical по contrast (gold text/ring) и, вероятно, по `html lang` (BUG-13) и form-label (BUG-14) — значит вердикт «PASS» по `41` Финальному гейту сейчас **недостижим** без правки `04`/`06_TOKENS`.
