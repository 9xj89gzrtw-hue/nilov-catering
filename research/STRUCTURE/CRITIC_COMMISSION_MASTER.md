## 2. КРОСС-ЭКСПЕРТНЫЙ P0-КОНСЕНСУС (найдено независимо ≥3 экспертами)

### P0-1. Контраст золота FAIL (WCAG) — A11Y(5.0), Visual(6.0), UIDesigner(5.5), Motion
`#B08D57` на `#FAF7F2` = **2.89:1** (пересчитано A11Y-экспертом). Используется как текст (overline, заголовки), акценты, разделители, **focus-ring** → FAIL 1.4.3 + 1.4.11. `gold-soft` ещё хуже — 2.11:1. При этом A3 в спеке ложно клеймит «контраст закрыт / AAA».
**Правка:** золото — только для НЕ-текстовых элементов (линии, рамки толщиной ≥3px, декор, иконки-не-текст) ИЛИ для текста на тёмном (ink) фоне. Для текста на ivory ввести `text-gold-ink` (gold на тёмном плитке) либо `ink`/`muted-foreground`. Поднять gold-text на ivory минимум до 4.5:1 (использовать затемнённый `#8A6D3B` или darker).

### P0-2. Бренд расколот: запрет «премиум» vs премиум-лексика в копирайте — Brand(6.0), CreativeDirector(4.8), Hospitality(3.0), UIDesigner(5.5), Marketing(3.4)
`01_VISUAL_DNA.md:25` заявляет «средний сегмент, НЕ премиум», но по всему `04_BLOCKS.md` размазан премиум/люкс-словарь (публичный тир «Премиум» `04:959`, «Гала на 500+», элит-логотипы, gold-как-статус). Нет Voice-документа, архетипа, UVP, Big Idea.
**Правка:** написать `40_BRAND_VOICE.md` (позиция «доступная премиальность» / «честный кейтеринг для любого бюджета»; tone-матрица do/don't; UVP в 1 строку; запрет элит-лексикона и ложных клиентских лого). Убрать слово «премиум» из публичных тиров → «Стандарт / Расширенный / Гала».

### P0-3. Фиктивный AI-брендинг без модели — AIUX(3.0), Frontend(5.0)
`AIConcierge` — только имя FAB-кнопки (`04:201`) и «optional-обёртка над /plan/helper» (`04:202`). Помощник `/plan/helper` — детерминированный wizard if/else (`04:937`), «рекомендации» — if/else по MIN_GUESTS. Прямое противоречие: `14:17` «AI-консьерж в резерве» ↔ `08:136` «AIConcierge НЕ в резерве». Имя «AI» при отсутствии модели = потенциально обманный премиум-брендинг (прозрачность 2/10).
**Правка:** до реальной модели убрать термин «AI» из публичного UI. Либо написать отдельный `XX_AI_CONCIERGE_SPEC.md` с моделью, fallback, прозрачностью, либо переименовать FAB в «Помощник» / «Консьерж».

### P0-4. SEO/домен: силос роутов и мёртвый sitemap (НЕ домен)

### P0-5. CMS = 0% хардкода, нет auth при `/account` — Frontend(5.0), Marketing(3.4), CMO-критик(3.0)
Весь контент (меню, цены, отзывы, блог) хардкод в `lib/data.ts`; CMS-слой из `04_BLOCKS.md:80-106` не реализован. `/account/orders`, `/partners`, `/subscribe` есть в IA, но **нет архитектуры auth/RBAC**. `catalog.json`/`lib/pricing.ts` — фактический SSOT, хотя спека claims «код не источник правды».
**Правка:** зафиксировать статус CMS как «не построена» в каноне; либо построить минимальный content-layer (Storyblok/Sanity+dev-Studio), либо официально перевести `lib/data.ts`+`lib/pricing.ts` в SSOT и описать data-access контракт. ### P0-6. Голые 🟡-факты, цена 950 vs 390 — Trust(3.5), Competitive(5.0), CRO(5.0), Brand(6.0), Copy(6.0)

Убрать элит-логотипы до верификации или вынести в «нам доверяют» с дисклеймером. Привести «18/19 лет»→«18 лет (с 2007)» единообразно.

### P0-7. Сервис не спроектирован («премиум» только как ценник) — Hospitality(3.0), Industry(6.0)

### P0-8. Hero-перегруз ~7-8 CTA при правиле «1 primary» — CRO(5.0), UIDesigner(5.5), ArtDirector(5.0), MobileUX(6.0), Landing(3.0)

## 3. P1-КЛАСТЕРЫ (найдены 2-3 экспертами, блокируют до приёмки)

- **Mobile bottom-nav мёртв** (MobileUX M1): `StickyMobileCTA` «всегда видима» + OR-правило → `MobileBottomNav` никогда не рендерится. + Chromium-only perf-детект (`effectiveType`/`deviceMemory`) не работает на Safari iOS (M4).
- **Motion-противоречия** (Motion B1-3, AnimationEng A7): parallax «на ВСЕХ карточках» (`01:17-19`,`04:359`) лоб в лоб с «только 1 hero-стрип» (`11:11,45`). Блокирующий прелоадер 1.4-1.9s (B4) ломает LCP-бюджет.
- **Lenis ≠ reduced-motion** (Performance P4-5, Interaction, Motion B9-10): 3 противоречащих disable-правила, нет централизованного гейта; Lenis по умолчанию на desktop = INP-риск.
- **IA: осиротевшие страницы** (IA Bug 7): ~9 страниц без места в nav/footer; `/contact` вне навигации; диеты вынесены в роуты наравне с форматами (каннибализация).
- **B2B-зрелость 2/10** (B2B B1-B11): форма не квалифицирует юрлицо (нет ИНН/роли/BANT), `calcTotal` слеп к НДС, нет account-based/SLA/KAM, повтор заказа — B2C-клон.
- **Отраслевое ядро провалено** (Industry I1-I16): нет холодовой цепи/маршрутизации, персонал не считается, `CHILD_RATE=0.6` — ложный прокси, гамма-скидка математически не даёт обещанные 15%, агрессивные скидки без политики депозита = маржинальный риск.
