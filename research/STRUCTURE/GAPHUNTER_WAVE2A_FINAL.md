## ВЕРДИКТ

**достигнута**: все ранее осутствовавшие страницы и сервисные узлы теперь
описаны и внутренне непротиворечивы. Четыре явных чек-листа пройдены:

- ✅ **Недостающие страницы есть (в spec):** `/delivery` (БЛОК 21),
  `/certificates` (БЛОК 22), `/reviews` (ренейм `/testimonials`, `02_IA`
  строка 82) — все прописаны с маршрутом, breadcrumbs и точкой входа из футера.
- ✅ **Служебные 404 / thank-you есть:** БЛОК 25 `ThankYouScreen` (`/thank-you`,
  `?source=`, noindex, router.push после fetch) + БЛОК 26 `NotFoundPage`
  (`/404`, глобальный `notFound()`, корректный HTTP 404, noindex).
- ✅ **Google-рейтинг РЕАЛЕН, не фейк:** БЛОК 24 `RatingBadge` — только живая
  ссылка на Google Business, `rating?`/`reviewsCount?` опц., прямой запрет
  хардкода «5.0»; плюс БЛОК 30 `AggregateRating` жёстко ЗАГЕЙЧЕН на
  `reviews.approved.length > 0` (строка 493). Фейк-схемы нет.
- ✅ **Мёртвый `/api/newsletter` задействован:** БЛОК 23 `NewsletterSignup`
  (email + чекбокс 152-ФЗ → `POST /api/newsletter`); API реально есть в репо
  (`15_REPO_AUDIT` §3). Закрывает старый баг «мёртвая ссылка».

**НО** полнота = 100% на бумаге, ~0% в коде (G-1). И чек-лист индустрии 2026

## ОСТАТОЧНЫЙ BUG-LIST (GapHunter, Wave 2А финал)

- **G-1 [КРИТ]** План ≠ код. Все роуты Волны 2А существуют только в
  `02_IA`/`04_BLOCKS`. Реальный `app/` всё ещё содержит легаси
  `/services`, `/services/weddings|corporate|private`, `/about`,
  `/testimonials`, `/offer`, `/pricing`, `/quote` и **НЕ содержит** ни
  `/delivery`, `/certificates`, `/thank-you`, `/404`, ни `/events`,
  `/menu/*`, `/plan/*`, `/reviews` (поиск по `*.tsx` → 0 совпадений).
  Полнота проверена на уровне IA; сборка не начата. Это доминирующий residual.

- **G-2 [ВАЖНО, RU-специфика]** `RatingBadge` (БЛОК 24) ссылается **только на
  Google**. Для петербургского B2B-кейтеринга основные площадки отзывов —
  **Яндекс.Карты/Бизнес** и **2ГИС**; Google вторичен (и частично
  недоступен в РФ). Охват реальных отзывов не закрыт. Добавить Yandex/2GIS
  RatingBadge (или единый блок «Отзывы о нас» со всеми площадками).

- **G-3 [СРЕДНЕ]** Нет механизма автоподтягивания реального рейтинга:
  `rating?`/`reviewsCount?` — опц. пропсы, заполняются вручную/плейсхолдером.
  Риск: при сборке дев вставит «5.0» как заглушку → de-facto фейк. Жёстко
  зафиксировать в spec: либо реальный pull (Google Places API / Yandex API),
  либо вариант «Оцените нас на …» БЕЗ числа. Проверить в код-ревью.

- **G-4 [СРЕДНЕ]** `SchemaBlock`-маппинг (БЛОК 30, таблица 9 строк) не
  покрывает новые страницы: `/delivery` (нужен `Service`+`Offer` по зонам
  доставки) и `/certificates` (комплаенс/`Organization`). `/thank-you` и
  `/404` — noindex, ок. Дописать маппинг.

- **G-5 [МЕЛКО]** Breadcrumbs явно зафиксированы для `/delivery` и
  `/certificates`, но **не** для `/reviews` (Главная › Отзывы) и `/seasonal`
  (Главная › Сезонное) — риск рассинхрона с правилом Breadcrumbs (БЛОК
  SiteHeader, строка 21).

- **G-6 [МЕЛКО, 2026-тренд]** Нет страницы политики доступности (`/accessibility`)
  и a11y-заявления. WCAG 2.2 AA заявлен во многих блоках, но страницы-подтверждения
  нет — в 2026 это элемент доверия/комплаенса.

- **G-7 [МЕЛКО]** `/thank-you` при прямом заходе без `?source` — не описано
  дефолтное состояние (generic-текст или редирект на `/`). Уточнить.

- **G-8 [МЕЛКО]** Careers/«Работа у нас» и Press/Media-kit — типичные страницы
  кейтеринг-сайта 2026; не в плане (низкий приоритет, но для B2B-доверия иногда
  нужны).

- **G-9 [НАПОМИНАНИЕ]** Даже при наличии страниц контент-claim'ы остаются
  `pending-verification` (TODO баг 2): номера сертификатов `/certificates`,
  реальный Google/Яндекс-рейтинг, «18 лет/3000+». При лончы страницы могут
  рендерить плейсхолдеры/«Оцените нас» — заложить в QA.
